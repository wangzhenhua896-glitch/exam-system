# 英语评分模块独立设计

> 版本：v1.0.0 | 日期：2026-04-13 | 分支：multi-fullscore-answers

## 1. 背景

英语科目评分逻辑原本散落在 3 个文件中（`three_layer_grader.py`、`qwen_engine.py`、`api_routes.py`），每次改英语逻辑需跨文件操作。重构后统一到独立模块 `app/english_grader.py`。

## 2. 模块结构

```
app/english_grader.py（~500行）
├── 文本处理工具
│   ├── normalize_english_text()    — 标准化：小写→去标点→去冠词→压缩空格
│   ├── detect_cjk()                — 检测中文字符
│   └── detect_pinyin()             — 拼音检测（支持白名单）
├── 采分点精确匹配
│   ├── _word_boundary_match()      — \b 单词边界匹配，避免 compass→compassion
│   ├── _match_scoring_point()      — 检查单个采分点是否命中
│   └── english_scoring_point_match() — 核心：预检→排除词→评分公式
├── LLM 兜底评分
│   ├── format_scoring_points_for_prompt() — 采分点 JSON → prompt 文本
│   ├── _get_english_fallback_system_prompt() — 兜底系统提示词
│   ├── grade_english_fallback()    — 调 LLM 判断语义等价
│   ├── _parse_english_output()     — 解析 LLM 输出
│   └── _calculate_confidence()     — 置信度计算
└── 主流程
    └── grade_english()             — 英语评分入口
```

## 3. 评分流程

```
grade_english(answer, question_answers, max_score, grading_engine, question)
  │
  ├─ 从 question_answers 提取 scope_type='scoring_point' 的采分点 JSON
  │
  ├─ 第1层：采分点精确匹配（english_scoring_point_match）
  │    ├─ 预检：CJK检测 → 词数检测 → 拼音检测
  │    ├─ 排除词表检查
  │    └─ 评分公式：
  │         ├─ max_hit_score：遍历采分点，取最高分的命中
  │         └─ hit_count：统计命中数，按 rules 查得分
  │
  ├─ 命中（score > 0）→ 直接返回，跳过 LLM，confidence=0.95
  │
  └─ 未命中 → LLM 兜底（grade_english_fallback）
       ├─ 用采分点 JSON 构造 prompt
       ├─ LLM 判断语义等价（3次重试）
       └─ 关键原则：LLM 只能决定"是否命中采分点"，不能创造额外给分理由
```

## 4. 调用关系

```
api_routes.py (/api/grade)
  └→ three_layer_grade()
       ├─ subject == 'english' → grade_english()    ← 本模块
       └─ subject != 'english' → 三层并行（关键词+向量+LLM）
```

`api_routes.py` 无需改动，调用链不变。

## 5. 采分点 JSON 结构

```json
{
  "id": "Q1",
  "text": "What is the other name of Chinese New Year?",
  "max_score": 2,
  "scoring_points": [
    {"id": "A", "score": 2, "keywords": ["spring festival"], "synonyms": []},
    {"id": "B", "score": 1, "keywords": ["spring", "festival"], "synonyms": []}
  ],
  "score_formula": "max_hit_score",
  "exclude_list": ["new year", "lunar new year"],
  "pinyin_whitelist": []
}
```

### 评分公式类型

**max_hit_score**：遍历采分点（按 score 降序），第一个命中的得分即为最终分。

**hit_count**：统计所有采分点命中数，按 rules 查表：
```json
{"type": "hit_count", "rules": [{"min_hits": 2, "score": 2}, {"min_hits": 1, "score": 1}]}
```

## 6. 返回值结构

`grade_english()` 返回与 `three_layer_grade()` 相同的 dict：

```python
{
    'final_score': float,           # 最终得分
    'strategy': 'max',              # 英语强制 max
    'layer_scores': {
        'keyword': float | None,    # 采分点匹配得分
        'vector': None,             # 英语跳过向量层
        'llm': float | None,        # LLM 兜底得分（命中时为 None）
    },
    'layer_details': {
        'keyword': {                # 采分点匹配详情
            'method': 'english_scoring_point',
            'sub_question_id': 'Q1',
            'hit_items': [...],
            'total_hits': int,       # hit_count 模式才有
            'scoring_items': [       # 每个采分点的命中状态
                {
                    'name': '采分点A: spring festival',
                    'score': 2,
                    'max_score': 2,
                    'hit': True,
                    'reason': '命中"spring festival"',
                    'matched_by': 'spring festival'
                }
            ],
            'comment': '命中采分点A (spring festival)，得2分。'
        },
        'vector': {'reason': 'skipped_for_english'},
        'llm': {...}
    },
    'llm_result': QwenGradingResult | None
}
```

## 7. 预检规则

| 顺序 | 检查 | 条件 | 结果 |
|------|------|------|------|
| 1 | CJK 检测 | 答案含中文字符 | 0 分 |
| 2 | 词数检测 | 标准化后词数 < 1 | 0 分 |
| 3 | 拼音检测 | 移除白名单后 >50% 词匹配拼音模式 | 0 分 |
| 4 | 排除词表 | 标准化后答案包含排除词 | 0 分 |

## 8. 已知限制

1. **多词关键词中间插词**：`light up fireworks` 不匹配 `light fireworks`，需显式加到 synonyms
2. **子串误匹配**：已用 `\b` 单词边界解决（`compass` 不会匹配 `compassion`）
3. **语义放松题依赖 LLM 兜底**：如第 3 题 Q3（嫦娥奔月），精确匹配可能 miss
4. **Q7-Q12 无采分点数据**：仍走通用 LLM 评分（用 rubric 脚本）

## 9. 涉及文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `app/english_grader.py` | 核心 | 英语评分完整模块 |
