# 题型编辑器扩展设计方案

> 版本: v1.0.0
> 日期: 2026-04-14
> 状态: 草案

## 1. 背景

当前系统只支持 essay（简答题）一种题型，前端通过 `englishEditCore.js` 的 5 步工作流编辑，后端通过三层并行评分（关键词+向量+LLM）。

DB 已有 `question_type` 列（取值：essay/single_choice/multi_choice/fill_blank/true_false/translation），但前端和评分引擎均未按题型分流。需要扩展支持选择题、填空题、判断题。

## 2. 设计原则

- **不新建表**：通过 `question_type` + `rubric.type` + `answer_text` 格式适配
- **确定性判分**：choice/fill_blank/true_false 不走 LLM，用规则精确判分
- **复用现有路由**：`editQuestion()` 函数中增加 `question_type` 分支
- **编辑器独立模块**：每种题型一个 JS 文件，通过 `window` 全局暴露

## 3. 架构总览

```
前端编辑器路由（question-bank.html）
├── question_type=essay       → englishEditCore.js（已有）
├── question_type=single_choice → choiceEditorCore.js（新建）
├── question_type=multi_choice  → choiceEditorCore.js（复用）
├── question_type=fill_blank    → fillBlankEditorCore.js（新建）
└── question_type=true_false    → choiceEditorCore.js（内嵌）

后端评分路由（api_routes.py /grade）
├── subject=english           → english_grader.py（已有）
├── question_type in choice   → choice_grader.py（新建）
├── question_type=fill_blank  → fill_blank_grader.py（新建）
└── 其他                      → three_layer_grader.py（已有）
```

## 4. 数据模型

### 4.1 choice 题型

**questions 表**：
- `question_type`: `single_choice` 或 `multi_choice`
- `content`: 题目文本
- `standard_answer`: 正确选项，如 `"A"`（单选）或 `"AC"`（多选）
- `rubric`: `{"type": "single_choice", "options": ["A","B","C","D"], "correct": "A"}`

**评分**：
- 单选：学生答案 == correct → 满分，否则 0 分
- 多选：全对满分，部分对 0 分（或按规则给部分分）

### 4.2 fill_blank 题型

**questions 表**：
- `question_type`: `fill_blank`
- `content`: 含 `___` 或 `____` 占位符的题目文本
- `standard_answer`: JSON 数组，如 `["经济基础", "上层建筑"]`
- `rubric`: `{"type": "fill_blank", "blanks": [{"index": 0, "answer": "经济基础", "score": 1, "synonyms": []}, ...]}`

**评分**：
- 逐空匹配：学生答案数组 vs 标准答案数组
- 支持 synonyms 同义词匹配（可选）
- 每空独立给分，求和得总分

### 4.3 true_false 题型

**questions 表**：
- `question_type`: `true_false`
- `content`: 题目文本
- `standard_answer`: `"正确"` 或 `"错误"`
- `rubric`: `{"type": "true_false", "correct": "正确"}`

**评分**：
- 学生答案 == correct → 满分，否则 0 分

### 5.2 choiceEditorCore.js 设计

通过 `window.ChoiceEditorCore` 暴露，接口与 `EnglishEditCore` 一致：

```javascript
window.ChoiceEditorCore = {
  useChoiceEditor: useChoiceEditor,  // 返回 { loadQuestion, saveAll, ... }
};
```

**useChoiceEditor() 返回的状态和方法**：
- `questionType` — `single_choice` / `multi_choice` / `true_false`
- `questionText` — 题目文本
- `options` — 选项列表 `[{label: 'A', text: '选项内容'}, ...]`
- `correctAnswer` — 正确选项（单选字符串，多选数组）
- `maxScore` — 满分值
- `loadQuestion(id)` — 从 DB 加载，解析 rubric 填充表单
- `saveAll()` — 构造 rubric JSON + 调用 update_question API
- `validate()` — 校验：至少 2 个选项、已选正确答案

**true_false 模式**：`questionType === 'true_false'` 时，选项固定为 `[{label: 'A', text: '正确'}, {label: 'B', text: '错误'}]`，不可编辑。

### 5.3 fillBlankEditorCore.js 设计

```javascript
window.FillBlankEditorCore = {
  useFillBlankEditor: useFillBlankEditor,
};
```

**useFillBlankEditor() 返回的状态和方法**：
- `questionText` — 含 `___` 占位符的题目文本
- `blanks` — 空位列表 `[{index, answer, score, synonyms: []}, ...]`
- `maxScore` — 满分值（各空 score 之和）
- `loadQuestion(id)` — 从 DB 加载，解析 content 中的 `___` 自动识别空位数
- `saveAll()` — 构造 rubric JSON + standard_answer JSON
- `validate()` — 校验：空位数与答案数一致、各空 score > 0

### 5.4 编辑器路由修改

`templates/question-bank.html` 的 `editQuestion()` 函数改为：

```javascript
const editQuestion = async (questionId) => {
    const qItem = questions.value.find(q => q.id === questionId);
    const qType = qItem.question_type || 'essay';

    if (qType === 'essay' && qItem.subject === 'english' && !qItem.parent_id) {
        // 英语简答题 → 英语编辑器（现有逻辑）
        currentPage.value = 'english-edit';
        await englishEdit.loadQuestion(questionId);
    } else if (['single_choice', 'multi_choice', 'true_false'].includes(qType)) {
        // 选择/判断题 → choiceEditor
        currentPage.value = 'choice-edit';
        await choiceEdit.loadQuestion(questionId, qType);
    } else if (qType === 'fill_blank') {
        // 填空题 → fillBlankEditor
        currentPage.value = 'fill-blank-edit';
        await fillBlankEdit.loadQuestion(questionId);
    } else {
        // 其他 → 通用表单编辑（现有逻辑）
        currentPage.value = 'question-manage';
    }
};
```

## 6. 后端实现

### 6.1 新增评分函数

**`app/choice_grader.py`**（~80 行）：
```python
def grade_choice(answer: str, rubric: dict, max_score: float) -> dict:
    """选择题/判断题确定性判分"""
    correct = rubric.get('correct', '')
    q_type = rubric.get('type', 'single_choice')

    if q_type == 'single_choice' or q_type == 'true_false':
        score = max_score if answer.strip().upper() == correct.strip().upper() else 0
    elif q_type == 'multi_choice':
        # 全对满分，部分对 0 分（可配置）
        score = max_score if set(answer.strip().upper()) == set(correct.strip().upper()) else 0

    return {
        'score': score,
        'confidence': 1.0,
        'comment': '正确' if score > 0 else f'正确答案是 {correct}',
        'details': {'correct_answer': correct, 'student_answer': answer},
        'model_used': 'rule',
    }
```

**`app/fill_blank_grader.py`**（~120 行）：
```python
def grade_fill_blank(answers: list, rubric: dict, max_score: float) -> dict:
    """填空题逐空匹配判分"""
    blanks = rubric.get('blanks', [])
    total = 0
    details = []
    for i, blank in enumerate(blanks):
        student_ans = answers[i] if i < len(answers) else ''
        correct_ans = blank['answer']
        synonyms = blank.get('synonyms', [])
        matched = student_ans.strip() == correct_ans.strip() or student_ans.strip() in synonyms
        score = blank['score'] if matched else 0
        total += score
        details.append({'index': i, 'correct': correct_ans, 'student': student_ans, 'score': score, 'matched': matched})

    return {
        'score': total,
        'confidence': 1.0,
        'details': {'blanks': details},
        'model_used': 'rule',
    }
```

question_type = question.get('question_type', 'essay')

if question_type in ('single_choice', 'multi_choice', 'true_false'):

## 7. 实现步骤（建议顺序）

### Phase 1：后端评分（确定性判分，不依赖前端）
1. `app/choice_grader.py` — 选择题/判断题判分函数
2. `app/fill_blank_grader.py` — 填空题判分函数

### Phase 2：前端编辑器
5. `static/js/choiceEditorCore.js` — 选择/判断编辑器
6. `static/js/fillBlankEditorCore.js` — 填空编辑器
7. `templates/question-bank.html` — 编辑器路由 + 页面区域
