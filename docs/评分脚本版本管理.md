# 评分脚本版本管理

## 概述

评分脚本（rubric_script）是单题评分的核心指令，直接决定 AI 阅卷的质量。每次脚本更新自动保留历史版本，支持回退，验证结果和场景变更都会联动到版本记录中。

## 设计思路

- 脚本存放在 `questions.rubric_script`，每次保存时旧版本不会丢失，自动归档到 `rubric_script_history` 表
- 版本号按题目独立递增（v1, v2, v3...），永不回退
- 回退操作本身产生新版本记录（如 v5: "回退到版本 2"），因此回退也可以撤销
- 新增测试场景（手动或批量生成）时自动记录版本快照
- 运行 verify-rubric 验证后，结果写入当前脚本对应的版本记录

## 数据库

### rubric_script_history 表

```sql
CREATE TABLE rubric_script_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id INTEGER NOT NULL,
    version INTEGER NOT NULL,
    script_text TEXT NOT NULL,
    avg_error REAL DEFAULT NULL,       -- 验证结果：平均误差
    passed_count INTEGER DEFAULT NULL, -- 验证结果：通过数
    total_cases INTEGER DEFAULT NULL,  -- 验证结果：总用例数
    note TEXT DEFAULT '',              -- 备注（如"初始版本"、"自动保存旧版本"、"新增场景"等）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (question_id) REFERENCES questions(id) ON DELETE CASCADE
);
CREATE UNIQUE INDEX idx_rsh_question_version
    ON rubric_script_history(question_id, version);
```

### 自动快照逻辑（update_question 内）

| 场景 | 行为 |
|------|------|
| 旧脚本非空且有变化 | UPDATE 前保存旧脚本为新版本（note="自动保存旧版本"） |
| 旧脚本为空，新脚本非空 | UPDATE 后保存新脚本为版本 1（note="初始版本"） |
| 脚本无变化 | 不产生版本记录 |

### 数据迁移

`init_database()` 末尾自动为已有脚本回填版本 1（幂等，重复启动不重复插入）：

```sql
INSERT INTO rubric_script_history (question_id, version, script_text, note)
SELECT id, 1, rubric_script, '迁移：已有脚本（初始版本）'
FROM questions
WHERE rubric_script IS NOT NULL AND rubric_script != ''
AND id NOT IN (SELECT question_id FROM rubric_script_history)
```

## API 端点

### 获取版本历史

```
GET /api/questions/<question_id>/script-history
```

返回该题所有版本，按 version DESC 排序。

**响应：**
```json
{
  "success": true,
  "data": [
    {
      "id": 8,
      "question_id": 5,
      "version": 3,
      "script_text": "...",
      "avg_error": 0.45,
      "passed_count": 5,
      "total_cases": 6,
      "note": "验证运行",
      "created_at": "2026-04-10 10:30:00"
    }
  ]
}
```

### 回退到指定版本

```
POST /api/questions/<question_id>/script-rollback
Body: {"version": 2}
```

**流程：**
1. 加载目标版本的脚本文本
2. 调用 `update_question()` 恢复（内部自动快照当前版本）
3. 更新最新版本记录的 note 为 "回退到版本 N"

**响应：**
```json
{
  "success": true,
  "data": {"restored_version": 2}
}
```

### 场景联动（自动触发）

以下端点新增测试用例后，自动在版本历史中记录：

| 端点 | 触发条件 | 记录内容 |
|------|----------|----------|
| `POST /api/questions/<id>/test-cases` | 手动新增单个场景 | note="新增场景（当前共N个场景）" |
| `POST /api/questions/<id>/generate-test-cases` | AI 批量生成场景 | note="批量新增N个场景（当前共M个场景）" |

### 验证联动（verify-rubric）

`POST /api/verify-rubric` 验证完成后，自动将结果写入当前脚本对应的历史版本：

- `avg_error`：所有用例的平均绝对误差
- `passed_count`：误差在容差内的用例数
- `total_cases`：有效用例总数

## 前端

### 题库页面（question-bank.html）

- 评分脚本区域旁有「版本历史」按钮（Clock 图标）
- 点击弹出版本历史表格：版本号、脚本预览、验证结果、备注、保存时间、操作
- 「预览」按钮：弹窗显示完整脚本内容
- 「回退」按钮：确认后回退（当前脚本自动保存，可撤销）

### 测试集页面（test-cases.html）

- 题目信息区显示当前脚本版本号 `vN`（在"已生成/未生成"标签旁）
