## 二、现有相关接口

### `/api/english/extract`（智能解析）
- 输入：`full_text`（完整原题文本）
- 功能：AI 提取子题 + 采分点 JSON
- 前端入口：英语编辑器 Step2 "智能解析"按钮

### `/api/english/extract-scoring-points`（采分点提取）
- 输入：`question_text` + `standard_answer` + `max_score`
- 功能：AI 从单个子题提取采分点 JSON

## 三、直接调用 `/api/english/extract` 的问题

### 问题1：original_text 是 HTML
`original_text` 含大量 `<p><span style="...">` 标签，直接传给 AI 会干扰解析，**需先剥离 HTML 标签**。

### 问题2：6 道题缺子题记录
ID 172-177 没有 Q1/Q2/Q3 子题行。评分引擎按子题评分，**子题不存在无法评分**，需先创建子题记录。

### 问题3：Prompt 未针对中文评分细则
`make_extract_prompt` 只让 AI 提取"reading material + sub-questions + scoring points"，而 `rubric_script` 中最有价值的**同义词、排除词、计分逻辑**都用中文写明了，AI 不一定能准确还原。

### 问题4：score_formula 格式可能不稳定
引擎要求严格格式：
- `"max_hit_score"`（字符串）
- `{"type": "hit_count", "rules": [{"min_hits": N, "score": S}]}`（对象）

AI 生成时格式可能偏差，导致引擎解析失败。

## 四、推荐方案：直接解析 rubric_script

`rubric_script` 已明确写出了所有需要的信息，用正则解析比让 AI 重新猜更可靠。

### rubric_script 结构规律（以 Chinese Tea Culture 为例）

```
第1题
问题：What is Longjing tea?
参考答案：A type of green tea.
采分点（关键词）：green tea
得分   条件
2分    写出 green tea（允许 a kind of green tea / it is green tea）
1分    只写出 tea 或 green（不完整）
0分    未写出 green tea；或出现拼音/汉字
同义表达可接受范围：
green tea → a green tea / green tea leaves
```

### 解析目标：生成标准采分点 JSON

```json
{
  "id": "Q1",
  "text": "What is Longjing tea?",
  "max_score": 2,
  "scoring_points": [
    {"id": "A", "score": 2, "keywords": ["green tea"], "synonyms": ["a green tea", "green tea leaves", "a kind of green tea", "it is green tea"]},
    {"id": "B", "score": 1, "keywords": ["tea", "green"], "synonyms": []}
  ],
  "score_formula": "max_hit_score",
  "exclude_list": [],
  "pinyin_whitelist": []
}
```

### 实施步骤

1. 对 ID 172-177 各创建 Q1/Q2/Q3 子题记录（`POST /api/questions`）
2. 解析各父题 `rubric_script`，提取每个子题的采分点
3. 将采分点 JSON 写入 `question_answers` 表（`POST /api/questions/{child_id}/answers`）
4. 补全 Chinese Tea Culture 的 Q2/Q3（ID 194/195）

### 涉及文件

| 文件 | 操作 |
|------|------|
| 新建脚本 `scripts/import_english_scoring_points.py` | 解析 + 批量导入 |
| `app/models/db_models.py` | 已有 CRUD，无需改动 |
| `app/api_routes.py` | 已有接口，无需改动 |

## 五、备选方案：AI 辅助（适合格式不规则时）

若某道题 `rubric_script` 格式差异较大，正则解析失败，可降级调用 `/api/english/extract-scoring-points`：

```
输入：question_text（子题问题）+ standard_answer（参考答案）+ rubric_script 片段
→ AI 生成采分点 JSON
→ 人工核对后写入
```

## 六、验证方式

导入完成后，对每道题各跑一次评分：
1. 满分答案 → 应得 2 分
2. 部分答案（只答一个关键词）→ 应得 1 分
3. 空白/汉字答案 → 应得 0 分
4. 同义表达 → 应得对应分数
