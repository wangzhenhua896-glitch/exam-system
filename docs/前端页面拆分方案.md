# 前端页面拆分方案

> 更新时间：2026-04-15
> 参考：`docs/前端代码审查与改进清单.md`

---

## 问题背景

`question-bank.html` 曾是一个 **3681 行的巨型单文件**，HTML 模板 + 内联 CSS + 内联 `<script>` 全部糅合在一起。核心矛盾：

- 单个 `setup()` 函数包含 45+ 个函数、60+ 个 ref/reactive 状态变量
- 题库列表、题目编辑、去重合并、导入导出、考试大纲等 7 个子页面全在一处
- 任何改动都需要在 3000+ 行中定位，维护成本极高

## 拆分策略

遵循 **"按路由拆页面，不按模块拆 JS"** 的原则：

- 不做 IIFE 模块化（两套模块系统并存增加复杂度）
- 不做 ES Module 迁移（`question-bank.html` 使用 `<script>` 全局引入，与 `dist/index.html` 的 ESM 体系不兼容）
- 每个功能域 = 独立 HTML 文件 + 独立 Flask 路由 + 自包含 Vue app
- 页面间通过 `window.location.href` + URL 参数传递状态

与已有页面一致：
- `/grading` → `dist/index.html`（独立评分页）
- `/test-cases` → `templates/test-cases.html`（独立测试集管理）
- `/sensitive-words` → `templates/sensitive-words.html`（独立敏感词管理）
- `/user-management` → `templates/user-management.html`（独立用户管理）

## 拆出理由

去重/合并是 `question-bank.html` 中 **代码量最大** 的模块（HTML 183 行 + JS 280 行 + 状态 14 个），逻辑复杂（两阶段去重 + 相似合并 + 逐字段选择），且与题目列表页的其他功能无交叉依赖。

### 模块边界

| 项目 | 内容 |
|------|------|
| **HTML 模板** | 触发按钮 × 2（183、186 行）+ 去重弹窗（251-427 行）≈ 183 行 |
| **JS 状态变量** | 14 个：`dedupLoading`, `dedupDialogVisible`, `dedupGroups`, `dedupFullscreen`, `dedupCurrentIndex`, `dedupDone`, `dedupFontSize`, `dedupPhase`, `dedupExactGroups`, `dedupSimilarGroups`, `exactKeepId`, `dedupTotalDuplicates`, `mergeMode`, `mergeLoading` |
| **JS 函数** | 13 个：`FIELD_DEFS`, `buildFieldRows`, `currentDedupGroup` (computed), `dedupProgress` (computed), `getFieldValue`, `findDuplicates`, `findSameNumber`, `confirmMerge`, `advanceToNext`, `confirmExactMerge`, `skipExactGroup`, `skipGroup`, `closeDedupDialog` |
| **外部依赖** | `loadQuestions()`, `loadStats()` — 合并/删除后刷新列表和统计 |
| **API 端点** | `POST /api/questions/find-duplicates`, `POST /api/questions/find-same-number`, `POST /api/questions/merge` |

### 改动方案

#### 1. 新建 `templates/dedup.html`

独立页面，包含：
- 去重弹窗的完整 HTML（去掉 `<el-dialog>` 外层，直接作为页面主体）
- 顶部导航栏（返回题库 + 标题）
- 完整 Vue setup()：14 个状态 + 13 个函数
- `onMounted` 自动触发 `findDuplicates()` 或 `findSameNumber()`（通过 URL 参数区分模式）

#### 2. URL 设计

| URL | 说明 |
|-----|------|
| `/dedup` | 默认去重（findDuplicates） |
| `/dedup?mode=merge` | 合并同题（findSameNumber） |

#### 3. 操作完成后跳转

合并/删除完成后，弹窗提示结果，点击"返回题库"跳转 `/management`。

#### 4. 修改 `question-bank.html`

- 题目列表的"去重处理"按钮 → `window.location.href = '/dedup'`
- 题目列表的"合并同题"按钮 → `window.location.href = '/dedup?mode=merge'`
- 删除 HTML：去重弹窗（251-427 行）
- 删除 JS：14 个状态 + 13 个函数（约 294 行）
- 从 `return` 语句中移除去重相关变量

#### 5. 修改 `app/app.py`

```python
@app.route("/dedup")
def dedup_page():
    from flask import send_file
    template_path = os.path.join(os.path.dirname(os.path.dirname(__file__)), 'templates', 'dedup.html')
    return send_file(template_path, mimetype='text/html')
```

## 设计原则

1. **每个页面 = 独立 HTML + 独立 Vue app** — 不共享 setup()，不做模块拆分
2. **页面间用 URL 参数传递状态** — `?id=xxx&mode=view`，不依赖 localStorage 传递业务数据
3. **保留已有模式** — `send_file` 返回静态 HTML，与 `/test-cases`、`/sensitive-words` 等一致
4. **英语编辑器不拆** — 它通过 `window.EnglishEditCore` 已经是独立模块，嵌入 question-bank 的是它的 UI 壳
5. **`viewQuestionDialog` 留在列表页** — 查看弹窗不需要跳转，只读模式才跳独立页
6. **拆出页面必须自包含** — 不依赖母页面的 ref/reactive 状态，通过 URL 或 API 获取数据
