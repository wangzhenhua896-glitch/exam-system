## 登录流程自动化

登录页面**不是用户名密码**，而是**选择科目进入系统**。

### Playwright 登录代码

```python
from playwright.sync_api import sync_playwright

def do_login(page, subject_label="英语"):
    """选择科目进入题库管理
    
    subject_label: "思想政治" | "语文" | "英语" 等
    """
    page.goto("http://localhost:5001/login")
    page.wait_for_timeout(2000)
    
    # 点击 el-select 打开下拉
    page.click('.el-select')
    page.wait_for_timeout(500)
    
    # 选择科目
    page.locator(f'.el-select-dropdown__item:has-text("{subject_label}")').click()
    page.wait_for_timeout(500)
    
    # 点击"进入系统"
    page.click('button:has-text("进入系统")')
    page.wait_for_timeout(5000)
    assert "management" in page.url, f"登录失败: {page.url}"
```

### 管理员入口

点击登录页底部的"管理员入口"可直接进入 `/admin`（科目管理、模型配置），**不需要选科目**，但没有题库管理功能。

### 切换科目

通过 `localStorage` 实现，无需重新登录：

```python
page.evaluate("() => localStorage.setItem('ai_grading_current_subject', 'general')")
page.reload()
page.wait_for_timeout(3000)
```

`localStorage` 中的关键 key：
- `ai_grading_current_subject` — 当前科目 (`politics`/`chinese`/`english`/`general`)
- `ai_grading_subjects` — 可选科目列表 JSON
- `ai_grading_teacher_name` — 教师名

## 科目与题目的关系

**重要发现**：`subject` 字段决定了编辑器类型。

| subject | 编辑器 | 子题支持 | 英语编辑器 |
|---------|--------|---------|-----------|
| `politics` | 普通编辑器 | 有 | 否 |
| `chinese` | 普通编辑器 | 有 | 否 |
| `general` | 普通编辑器 | 有 | 否 |
| `english` | **英语编辑器** | 无（State B） | 是 |

**英语编辑器入口条件**（`question-bank.html:3302`）：
```javascript
if (qItem && qItem.subject === 'english' && !qItem.parent_id) {
    currentPage.value = 'english-edit';
    await englishEdit.loadQuestion(questionId);
    return;
}
```

即：`subject === 'english'` 且 `parent_id === null` 才进入英语编辑器。

**当前数据库状态**（2026-04-14）：
- `subject=general` (Q166-Q171): 有子题（各3个），但不进英语编辑器
- `subject=english` (Q172-Q177): 无子题，能进英语编辑器（State B 模式）

## API 测试技巧

### 基础请求

```python
import requests, json

BASE = "http://localhost:5001"

def api_get(path):
    r = requests.get(f"{BASE}{path}")
    return r.json()

def api_put(path, data):
    r = requests.put(f"{BASE}{path}", json=data)
    return r.json()
```

### 获取题目

```python
# 单题
data = api_get("/api/questions/166")
q = data.get("data", data)

# 子题列表
children = api_get("/api/questions/166/children")
items = children.get("data", children)

# 按科目筛选
questions = api_get("/api/questions?subject=english")
```

### rubric 字段类型注意

API 返回的 `rubric` 字段是 **dict**（不是 JSON string）：
```python
rubric = q.get("rubric")  # 已经是 dict
# 不需要 json.loads()
```

但 `rubric_script` / `standard_answer` 等字段是 **string** 或 **None**。

### 重置 workflow_status

测试前重置工作流状态：

```python
api_put("/api/questions/166/workflow-status", {
    "workflow_status": json.dumps({
        "current_step": "per_question",
        "active_question_index": 0,
        "question_status": {},
        "completed_steps": []
    })
})
```

### Round-trip 测试

```python
# 读取原始数据
q_before = api_get("/api/questions/190")
q = q_before.get("data", q_before)
original_rubric = q.get("rubric")
if isinstance(original_rubric, dict):
    original_rubric = json.dumps(original_rubric)

# 修改并保存
test_rubric = json.dumps({"type": "essay", "test_marker": "XYZ"})
r = requests.put(f"{BASE}/api/questions/190", json={
    "rubric": test_rubric,
    "parent_id": 170,  # 必须带，否则会被清空
    "max_score": 2.0,
    "content": q.get("content", ""),
    "standard_answer": q.get("standard_answer") or "",
})

# 验证写入
q_after = api_get("/api/questions/190")
q2 = q_after.get("data", q_after)
saved = q2.get("rubric")
# saved 是 dict，不是 string
assert isinstance(saved, dict) and saved.get("test_marker") == "XYZ"

# 还原
requests.put(f"{BASE}/api/questions/190", json={
    "rubric": original_rubric,
    "parent_id": 170,
    "max_score": 2.0,
    "content": q.get("content", ""),
    "standard_answer": q.get("standard_answer") or "",
})
```

**关键**：PUT 请求必须带 `parent_id`，否则后端 `update_question()` 全量更新会把 `parent_id` 设为 NULL。

## 英语编辑器测试要点

### 进入英语编辑器

1. 登录选择"英语"科目（或切 `localStorage` 到 `english`）
2. 题目列表中找到 `subject=english` 的题
3. 点击"编辑"按钮（不是"英语编辑"按钮 — 没有这个按钮）
4. 代码自动判断 `subject === 'english'` 进入英语编辑器

### State A vs State B

- **State A**: 有子题（`isStateA=true`），直接进入 `per_question` 步骤
- **State B**: 无子题（`isStateA=false`），进入 `extract` 步骤（显示阅读材料 + 粘贴区）

### 子题编辑区检查

```python
# 检查 textarea 可见
textareas = page.locator('textarea:visible, .el-textarea__inner:visible')
assert textareas.count() > 0, "没有可见编辑区"

# 检查子题导航
q2_nav = page.locator('button:has-text("Q2")').first
assert q2_nav.count() > 0 and q2_nav.is_visible()
```

### 确认子题 → 进入下一步

```python
for qi in range(1, 4):
    page.locator(f'button:has-text("Q{qi}")').click()
    page.wait_for_timeout(800)
    confirm = page.locator('button:has-text("确认完成"), button:has-text("确认")').first
    if confirm.count() > 0 and confirm.is_visible():
        confirm.click()
        page.wait_for_timeout(800)
```

## 数据 round-trip 测试

验证"写入 → 读取 → 数据一致"的完整流程。

### 测试模式

1. **读取原始值** → 保存到变量
2. **写入测试值** → PUT 请求
3. **重新读取** → 验证测试值存在
4. **还原原始值** → PUT 请求

### 必须验证的字段

| 字段 | 检查点 |
|------|--------|
| `rubric` | dict 格式，包含预期 key |
| `parent_id` | 写入后不丢失（全量更新陷阱） |
| `rubric_script` | 子题级别非空 |
| `scoring_strategy` | 从 rubric 中正确提取 |
| `standard_answer` | 不被清空 |

### parent_id 丢失问题

后端 `update_question()` 是全量更新。如果 PUT 请求不带 `parent_id`，会被设为 NULL：

```python
# 错误：parent_id 会被清空
requests.put(f"{BASE}/api/questions/190", json={
    "rubric": new_rubric,
    "content": "...",
})

# 正确：必须带 parent_id
requests.put(f"{BASE}/api/questions/190", json={
    "rubric": new_rubric,
    "parent_id": 170,  # 关键！
    "content": "...",
})
```

## 常见坑

### 1. rubric 是 dict 不是 string

```python
# 错误
rubric = q.get("rubric")
assert "keyword" in rubric  # TypeError if string

# 正确
rubric = q.get("rubric")
if isinstance(rubric, dict):
    assert "keyword" in rubric
elif isinstance(rubric, str):
    assert "keyword" in rubric
```

### 2. rubric_script 可能是 None

```python
rs = c.get("rubric_script")
rs_len = len(rs) if rs else 0
# 不能用 len(c.get("rubric_script") or "") — 会在 dict 时报错
```

### 3. f-string 中对 dict 不能 slice

```python
# 错误
f"rubric={rubric[:100]}"  # TypeError: unhashable type: 'slice'

# 正确
f"rubric={json.dumps(rubric, ensure_ascii=False)[:100]}"
```

### 4. wait_for_url 带时间戳参数

URL 格式：`/management?t=1776177101033`，`**/management` 匹配不上。

```python
# 错误
page.wait_for_url("**/management", timeout=10000)

# 正确
page.wait_for_timeout(5000)
assert "management" in page.url
```

### 5. el-select 的 input 是 readonly

```python
# 错误 — readonly 元素不能 fill
page.fill('input[type="text"]', 'admin')

# 正确 — 点击下拉框选择
page.click('.el-select')
page.locator('.el-select-dropdown__item:has-text("选项")').click()
```

### 6. 科目决定编辑器类型

测试英语编辑器功能时，必须确保题目 `subject=english`。`subject=general` 的题目即使内容是英语，也不会进入英语编辑器。

## 自动化测试脚本模板

```python
#!/usr/bin/env python3
"""测试脚本模板"""
import json, sys, requests
from playwright.sync_api import sync_playwright

BASE = "http://localhost:5001"
passed = failed = 0
results = []

def check(name, condition, detail=""):
    global passed, failed
    if condition:
        passed += 1
        results.append(f"  PASS  {name}")
    else:
        failed += 1
        results.append(f"  FAIL  {name}  -- {detail}")

def api_get(path):
    return requests.get(f"{BASE}{path}").json()

def do_login(page, subject="英语"):
    page.goto(f"{BASE}/login")
    page.wait_for_timeout(2000)
    page.click('.el-select')
    page.wait_for_timeout(500)
    page.locator(f'.el-select-dropdown__item:has-text("{subject}")').click()
    page.wait_for_timeout(500)
    page.click('button:has-text("进入系统")')
    page.wait_for_timeout(5000)

# === 测试逻辑 ===
# ...

# === 汇总 ===
for r in results:
    print(r)
print(f"\n通过: {passed}/{passed+failed}")
sys.exit(1 if failed > 0 else 0)
```
