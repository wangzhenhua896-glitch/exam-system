## 二、发现并修复的 Bug

### Bug 1: 加载已有题目时子题无法确认完成（P1）✅ 已修复

**现象**：编辑已有英语题目，Q2/Q3 的"确认完成"按钮始终 disabled。即使用户手动填写了标准答案也无法操作。

**根因**：三层问题叠加：
1. Word 导入的子题 `standard_answer` 在 DB 中为 null
2. `loadChildren()` 信任 workflow_status 中的 `completed` 状态，但数据实际不完整
3. 恢复的 `activeQuestionIndex` 指向已完成的题（如 Q3），而非第一个待配置的题

**修复**（`englishEditCore.js`）：
1. `loadChildren()` 中增加数据完整性校验：`status === 'completed' && !isQuestionComplete(sq)` → 重置为 `pending`
2. 恢复活跃题索引后，确保指向一个非 completed 的题；若没有任何 completed 的题，从第一题开始
3. 新增 `confirmSubQuestion(qi)` 方法，支持展开修改后重新确认，且自动推进到下一题

### Bug 2: 步骤指示器圆圈未渲染 ⏭️ 非 Bug

**分析**：步骤指示器使用自定义 div + inline style 实现（非 `el-steps` 组件），浏览器中正常显示。Playwright 查找 `.el-step` 返回空是选错了选择器。

### Bug 3: 展开子题后 textarea 累积（P2）✅ 已修复

**现象**：Q2 展开后有 2 个 textarea（Q1 未折叠），Q3 展开后有 3 个。

**修复**：新增 `collapseAllSubQuestions()` 函数，在 `completeCurrentQuestion()`、`confirmSubQuestion()` 和 `goToQuestion()` 中调用，确保同一时间只有一个子题编辑区域可见。

### Bug 4: "进入下一步"按钮不出现（P2）✅ 已修复

**根因**：与 Bug 1 联动——子题 status 未正确流转，`allQuestionsCompleted()` 返回 false。

**修复**：随 Bug 1 一同修复。`confirmSubQuestion()` 自动推进活跃题索引，最终 `allQuestionsCompleted()` 返回 true。
