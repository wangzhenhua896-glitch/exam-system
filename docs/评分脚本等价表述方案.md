# 评分脚本等价表述增强方案

## 问题现状

当前 prompt 已经要求模型在每个得分点列出"【必含关键词】"和"【等价表述】"，但实际生成的脚本质量不稳定：
- 有些得分点只列了 1 个关键词，不够覆盖表述差异
- 等价表述经常遗漏常见同义表达
- 不同 prompt 版本要求不一致（System Prompt 严格，User Prompt 笼统）
- 中文 `RUBRIC_SCRIPT_SYSTEM_PROMPT` 列了 7 个部分（缺独立【反作弊规则】），CLAUDE.md 要求 8 个

### 当前 prompt 体系

| Prompt | 文件:行 | 关键词要求 | 等价表述要求 | 结构要求 |
|--------|---------|-----------|-------------|---------|
| 中文 System | api_routes.py:90 | "每个得分点必须列出【必含关键词】" | "每个得分点必须列出【等价表述】（至少 3 个）" | 7 部分（缺独立反作弊） |
| 中文 User | api_routes.py:2386 | "包含关键词和等价表述辅助判断"（笼统） | 同左 | 未明确 |
| 英语 System | english_prompts.py:12 | `[Required Keywords]` (at least 2) | `[Equivalent Expressions]` (at least 3) | 8 部分 |
| 英语 User | english_prompts.py:224 | "Keywords and equivalent expressions" | 同左 | 未明确 |
| 英语编辑器 | english_prompts.py:685 | "Key answer points with keywords and equivalent expressions" | 同左 | 简化版 |
| 批量出题 | auto_generate.py:179 | "关键词 + 等价表述" | 同左 | 5 部分（简化） |

### 阅卷 prompt 中的指示

`qwen_engine.py` 的阅卷 prompt 已经告诉模型如何使用等价表述：
```
【匹配规则】
1. 【必含关键词】必须出现在学生答案中 → 得分
2. 如果必含关键词未出现，检查【等价表述】中是否有任意一个命中 → 得分
3. 关键词匹配优先于语义匹配
```

但实际脚本质量不一，有些脚本的等价表述就写了一两个，匹配效果差。

## 方案

### 核心改动：统一和强化 prompt 中的等价表述要求

不需要改评分引擎代码，只需要改进 3 个 System Prompt 和 2 个 User Prompt。

### 改动 1：中文 System Prompt 补全 8 部分结构

**文件**: `app/api_routes.py`，`RUBRIC_SCRIPT_SYSTEM_PROMPT`

当前第 3 部分之后直接跳到"作答情况分类"，缺独立的【反作弊规则】。

在【逐项评分规则】之后、【作答情况分类】之前，插入：

```
4. 【反作弊规则】
   明确以下情况直接判 0 分：
   - 复制题干内容作为答案
   - 答案为空或仅含标点
   - 答非所问（内容与题目完全无关）
   - 照抄标准答案以外的无关文本
```

这样结构变为 8 部分，与 CLAUDE.md 和自查方案对齐。

### 改动 2：强化等价表述的数量和质量要求

在中文 System Prompt 的【逐项评分规则】部分，将现有的：
```
每个得分点必须列出【必含关键词】和【等价表述】（至少 3 个）。
```

改为更具体的要求：
```
每个得分点必须同时列出：
【必含关键词】至少 2 个，优先选择核心术语和专业名词。
【等价表述】至少 5 个，需覆盖以下维度：
  - 同义替换（如"促进"→"推动"/"助推"）
  - 上下位词（如"市场调节"→"价格机制"/"供求机制"）
  - 口语化表达（如"资源分配"→"把资源分到需要的地方"）
  - 部分匹配（如"发挥市场作用"→"市场"/"市场化"单独出现也算）
  - 反面表述的否定形式（如"不是计划经济"→"非计划"/"不靠政府分配"）
```

### 改动 3：中文 User Prompt 增加等价表述显式要求

**文件**: `app/api_routes.py`，`generate_rubric_script` 函数中的 user_prompt

当前：
```
4. 包含关键词和等价表述辅助判断
```

改为：
```
4. 每个得分点必须包含【必含关键词】（至少2个）和【等价表述】（至少5个，覆盖同义词/口语化/上下位/部分匹配等多个维度）
```

### 改动 4：英语 System Prompt 维持现状

英语 System Prompt（`english_prompts.py:12`）已经比较严格（`at least 2` / `at least 3`），但可以同步提升：

```
[Required Keywords] — at least 2 per point, prioritize domain-specific terms
[Equivalent Expressions] — at least 5 per point, covering:
  - Synonyms (e.g., "analyze" → "examine", "evaluate")
  - Paraphrases (e.g., "improve efficiency" → "make things work better")
  - Partial matches (e.g., "economic growth" → "growth" alone counts)
  - Related concepts (e.g., "supply and demand" → "market forces")
```

### 改动 5：auto_generate.py 简化版 prompt 对齐

**文件**: `auto_generate.py`，`RUBRIC_SCRIPT_SYSTEM`

补充等价表述的具体要求，与主 System Prompt 对齐。

## 改动清单

| 文件 | 改动内容 | 影响范围 |
|------|---------|---------|
| `app/api_routes.py` | `RUBRIC_SCRIPT_SYSTEM_PROMPT`：补反作弊规则（7→8部分），强化等价表述要求（3→5个+维度） | /api/generate-rubric-script |
| `app/api_routes.py` | `generate_rubric_script` user_prompt：等价表述要求具体化 | 同上 |
| `app/english_prompts.py` | `RUBRIC_SCRIPT_SYSTEM_PROMPT_EN`：等价表述从 at least 3 → at least 5，加维度说明 | 英语评分脚本生成 |
| `app/english_prompts.py` | `make_rubric_script_prompt_en`：确保项第4条具体化 | 同上 |
| `auto_generate.py` | `RUBRIC_SCRIPT_SYSTEM`：补充等价表述具体要求 | 批量自动出题 |

## 不改的东西

- **不改评分引擎代码**：`qwen_engine.py` 的阅卷 prompt 已经正确指示如何使用等价表述
- **不改数据库结构**：评分脚本是文本，不需要额外字段
- **不改前端**：生成脚本的 UI 交互不变
- **不改英语编辑器**：采分点的 synonyms 是结构化数据，与脚本层面的等价表述是不同层面

## 验证

1. 用同一道题目重新生成评分脚本，对比改动前后等价表述的数量和覆盖维度
2. 用标准答案测试用例跑 verify-rubric，确认准确率不下降
3. 检查生成的脚本是否包含独立的【反作弊规则】章节
