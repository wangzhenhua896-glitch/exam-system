# 页面设计规范

## 技术约束

- Vue 3 CDN + Element Plus，**非 npm 构建**，无 `.vue` SFC
- 单文件 `.html`，`<template>` + `<style>` + `<script>` 全部在一个文件内
- Vue 使用 Options API 的 `setup()` 函数风格（`ref`、`computed`、`watch`、`onMounted`）
- JS 语法检查：`node --check /tmp/check.js`（先从 HTML 中提取 `<script>` 内容）

## 页面结构

### 整体布局（三栏）

```
┌─────────────────────────────────────────────────┐
│  Header（logo + 状态标签 + 右侧信息）              │
├────────┬────────────────────────────────────────┤
│        │                                        │
│ Sidebar│  Content（主内容区，可滚动）               │
│ 160px  │                                        │
│        │                                        │
└────────┴────────────────────────────────────────┘
```

- `.layout`：`height: 100vh; display: flex; flex-direction: column`
- `.main-container`：`display: flex; flex: 1; overflow: hidden`
- `.sidebar`：固定 160px，白色背景，右侧边框
- `.content`：`flex: 1; padding: 20px; overflow-y: auto; background: #f5f7fa`

### 页面切换

用 `v-if` + `currentPage` ref 实现 SPA 路由，不引入第三方路由库。

```js
const currentPage = ref('overview');
function handleMenuSelect(index) { currentPage.value = index; }
```

### Sidebar 分组

```html
<div class="menu-group-title">题库管理</div>
<el-menu :default-active="currentPage" @select="handleMenuSelect">
    <el-menu-item index="overview"><el-icon><DataAnalysis /></el-icon><span>题库总览</span></el-menu-item>
    ...
</el-menu>
```

## 工具栏

列表页工具栏分两行：**筛选行**（左对齐）+ **操作行**（右对齐）。

### 筛选行

```html
<div style="display: flex; align-items: center; gap: 12px; margin-bottom: 12px; flex-wrap: wrap;">
    <!-- 科目由登录时选择，通过 localStorage 读取，页内不再重复选择 -->
    <el-input v-model="searchKeyword" placeholder="搜索..." clearable style="width: 280px;">
        <template #prefix><el-icon><Search /></el-icon></template>
    </el-input>
    <span style="color: #909399; font-size: 13px; margin-left: auto;">共 {{ total }} 题</span>
</div>
```

### 操作行

```html
<div style="display: flex; justify-content: flex-end; gap: 8px; margin-bottom: 16px;">
    <el-button size="small" type="primary" plain :loading="xxxLoading">
        <el-icon><Upload /></el-icon> 导入
    </el-button>
    ...
</div>
```

### 按钮规范

| 场景 | style | 说明 |
|------|-------|------|
| 主操作 | `type="primary"` | 保存、确认、导入 |
| 次操作 | `type="primary" plain` | 导入、导出、批量操作 |
| 危险操作 | `type="danger" plain` | 删除、去重 |
| 成功操作 | `type="success" plain` | 导出、验证通过 |
| 表格内操作 | `size="small" plain` | 查看、编辑、删除 |
| 弹窗内 link | `type="primary" link` | 预览、回退等文字链接 |

所有按钮统一 `size="small"`。异步操作必须加 `:loading="xxxLoading"`。

## 表格

### 基本结构

```html
<el-table :data="paginatedXxx" stripe border style="width: 100%">
    <el-table-column prop="xxx" label="xxx" width="120" />
    <el-table-column label="操作" width="300" fixed="right">
        <template #default="scope">
            <el-button type="primary" size="small" plain @click="view(scope.row.id)">查看</el-button>
            <el-button type="warning" size="small" plain @click="edit(scope.row.id)">编辑</el-button>
            <el-popconfirm title="确定删除？" @confirm="remove(scope.row.id)">
                <template #reference><el-button type="danger" size="small" plain>删除</el-button></template>
            </el-popconfirm>
        </template>
    </el-table-column>
</el-table>
```

- 长文本列设 `width`，操作列 `fixed="right"`
- 删除必须用 `el-popconfirm` 二次确认
- 分页用 `el-pagination`，与 `paginatedXxx` computed 联动

## el-collapse 折叠面板

### 折叠表单区域

用 `el-collapse` 包裹表单分区，控制默认展开/折叠：

```html
<el-collapse :model-value="viewMode === 'create' ? ['1'] : []" style="margin-bottom: 12px;">
    <el-collapse-item title="基本信息" name="1">
        <!-- 表单内容 -->
    </el-collapse-item>
</el-collapse>
```

- 用 `:model-value` 数组控制哪些面板默认展开（不能用 `v-model`，因为不需要用户展开状态）
- **新建模式默认展开**基本信息和题干，**编辑/查看模式默认折叠**基本信息
- 原题（原始未处理）始终默认折叠

### 折叠面板标题内嵌操作（仅限非关键操作）

折叠面板标题里可以放操作链接，但**只有该操作在折叠状态也能被用户从其他入口触达时才这么做**。如果某个操作没有其他入口，放在折叠标题里 = 用户永远发现不了。

```html
<el-collapse-item name="orig">
    <template #title>
        <span>原题（原始未处理）</span>
        <!-- 仅放非核心的提示性信息，如"未填写"状态标签 -->
    </template>
</el-collapse-item>
```

**反例（不要这样做）：** 把"一致检查"放在默认折叠的"原题"标题里 → 用户看不到按钮，以为没有这个功能。

## 表单标签内嵌操作（核心设计原则）

**设计原则：辅助操作不占行、字号可调、突出内容。适合中老年视力。**

### 四条子原则

1. **辅助操作不占行** — 放在字段 label 右侧，用 12px 灰/蓝色文字链接，不单独占一行或一个按钮位。节省垂直空间给内容
2. **可操作按钮不能藏在折叠面板里** — 默认折叠的面板里放操作按钮 = 用户看不到。辅助操作必须放在始终可见的区域
3. **辅助操作跟随关联字段** — 按钮放在它操作的字段 label 上，如"自动生成"放在"分数分布"label 上，"一致检查"放在"题干"label 上
4. **字号可调，适应不同视力** — 所有文本密集区域（textarea、弹窗正文）必须提供 A-/A+ 字号控制，默认 16px，范围 12-24px。中老年用户可放大到舒适大小，而不影响页面布局（因为控制在 label 里不占行）

用 `<template #label>` 在表单标签右侧放置辅助链接：

```html
<el-form-item>
    <template #label>
        <span>字段名</span>
        <a href="#" @click.prevent="action"
           :style="{ marginLeft: '12px', fontSize: '12px', color: loading ? '#c0c4cc' : '#409eff', textDecoration: 'none' }">
            {{ loading ? '处理中...' : '操作名' }}
        </a>
    </template>
    <el-input ...></el-input>
</el-form-item>
```

### 内嵌操作清单

| 字段 | 操作 | 颜色 | 说明 |
|------|------|------|------|
| 题干（处理后） | 一致检查 | `#409eff`（加载中 `#c0c4cc`） | 检查原题与题干/答案/规则一致性 |
| 题干（处理后） | A- / A+ 字号控制 | `#909399` | 控制所有 textareas 字号 |
| 标准答案 | 清除分数标注 | `#e6a23c` | 清除答案中的分数标记 |
| 分数分布 | 自动生成 | `#409eff`（加载中变 `#c0c4cc`） | AI 生成得分点 |

### 字号控制

```html
<span style="margin-left: 12px; font-size: 12px; color: #c0c4cc;">
    <a href="#" @click.prevent="formFontSize = Math.max(12, formFontSize - 2)" style="color: #909399; text-decoration: none;">A-</a>
    <span style="margin: 0 2px;">{{ formFontSize }}</span>
    <a href="#" @click.prevent="formFontSize = Math.min(24, formFontSize + 2)" style="color: #909399; text-decoration: none;">A+</a>
</span>
```

- 范围 12-24px，步长 2，**默认 16px**（适合中老年阅读）
- 绑定到所有 textarea：`:style="{ fontSize: formFontSize + 'px' }"`
- 复用到弹窗内：弹窗用独立的 `dedupFontSize`，放在弹窗 header 里
- 字号控制本身始终 12px（它不是内容，不需要放大）
- 用户调整字号后，控件不占额外空间，始终在 label 同一行

## 弹窗（el-dialog）

### 简单弹窗

```html
<el-dialog v-model="visible" title="标题" width="700px" :close-on-click-modal="true">
    <!-- 内容 -->
</el-dialog>
```

### 复杂弹窗（带 header 工具栏）

需要字号控制 + 全屏切换时，用 `#header` slot：

```html
<el-dialog v-model="visible" :width="fullscreen ? '100%' : '900px'" :fullscreen="fullscreen" :close-on-click-modal="false">
    <template #header="{ titleId, titleClass }">
        <div style="display: flex; align-items: center; justify-content: space-between; width: 100%;">
            <span :id="titleId" :class="titleClass">{{ dynamicTitle }}</span>
            <div style="display: flex; gap: 6px; align-items: center;">
                <el-button size="small" @click="fontSize = Math.max(12, fontSize - 2)">A-</el-button>
                <span style="font-size: 12px; color: #909399; min-width: 32px; text-align: center;">{{ fontSize }}px</span>
                <el-button size="small" @click="fontSize = Math.min(24, fontSize + 2)">A+</el-button>
                <el-button size="small" @click="fullscreen = !fullscreen">
                    {{ fullscreen ? '退出全屏' : '全屏' }}
                </el-button>
            </div>
        </div>
    </template>
    <div :style="{ fontSize: fontSize + 'px' }">...</div>
</el-dialog>
```

- 涉及数据修改的弹窗用 `:close-on-click-modal="false"` 防误关
- 弹窗内标题根据模式动态切换（如去重 vs 合并同题）

### 弹窗内确认

合并/删除等不可逆操作，在弹窗内再用 `ElMessageBox.confirm` 二次确认：

```js
ElMessageBox.confirm('确认合并？此操作不可撤销', '提示', {
    confirmButtonText: '确认',
    cancelButtonText: '取消',
    type: 'warning'
}).then(() => { /* 执行 */ }).catch(() => {});
```

## 异步操作模式

所有异步操作遵循同一模式：

```js
const xxxLoading = ref(false);

const doXxx = async () => {
    // 前置校验
    if (!field.value.trim()) {
        ElMessage.warning('请先填写 xxx');
        return;
    }
    xxxLoading.value = true;
    try {
        const { data } = await axios.post('/api/xxx', { ... });
        if (data.success) {
            // 更新本地状态
            ElMessage.success('操作成功');
        } else {
            ElMessage.error('失败：' + (data.error || '未知错误'));
        }
    } catch (e) {
        ElMessage.error('失败：' + (e.response?.data?.error || e.message));
    } finally {
        xxxLoading.value = false;
    }
};
```

- `ref` 命名：`xxxLoading`
- 按钮绑定：`:loading="xxxLoading"`
- 内嵌链接文本：`{{ xxxLoading ? '处理中...' : '操作名' }}`
- 内嵌链接颜色：加载中变 `#c0c4cc`，正常为 `#409eff`

## JS 语法验证

编辑 `<script>` 后必须验证语法：

```bash
# 1. 从 HTML 提取 JS
python3 -c "
import re
with open('templates/xxx.html') as f:
    html = f.read()
match = re.search(r'<script>(.*?)</script>', html, re.DOTALL)
with open('/tmp/check.js', 'w') as f:
    f.write(match.group(1))
"
# 2. 检查语法
node --check /tmp/check.js
```

常见坑：
- `Vue.computed(() => { ... });` 的闭合 `});` 不能被 `.sort()` 的 `});` 代替
- 多层嵌套的箭头函数容易少闭合括号

```html
<el-row :gutter="16" style="margin-bottom: 24px;">
    <el-col :span="6">
        <el-card shadow="hover" style="text-align: center;">

            <div style="color: #909399; margin-top: 4px;">标签</div>
        </el-card>

```

```html

```
