# 英语采分点精确匹配评分 — 实现记录

> 版本：v1.0.0 | 日期：2026-04-13 | 分支：multi-fullscore-answers

## 实现概述

将英语评分从"全靠LLM"改为"采分点精确匹配为主 + LLM 兜底"。
采分点命中时确定性给分，跳过 LLM 调用；未命中时 LLM 用采分点 JSON 判断语义等价。

## 数据模型

### DB 存储结构

**子题**（questions 表，parent_id 指向父题）：
```
id=178, parent_id=166, question_number="Q1", title="Chinese New Year - Q1",
max_score=2, scoring_strategy="max_hit_score", subject="english"
```

**采分点 JSON**（question_answers 表，scope_type='scoring_point'）：
```json
{
  "id": "Q1",
  "text": "What is the other name of Chinese New Year?",
  "max_score": 2,
  "scoring_points": [
    {"id": "A", "score": 2, "keywords": ["spring festival"], "synonyms": []},
    {"id": "B", "score": 1, "keywords": ["spring", "festival"], "synonyms": []}
  ],
  "score_formula": "max_hit_score",
  "exclude_list": ["new year", "lunar new year"],
  "pinyin_whitelist": []
}
```

hit_count 类型的 score_formula：
```json
{"type": "hit_count", "rules": [{"min_hits": 2, "score": 2}, {"min_hits": 1, "score": 1}]}
```

## 评分流程

```
学生答案
  ↓
第1层：采分点精确匹配（english_scoring_point_match）
  ├─ 命中（score > 0）→ 直接返回，跳过 LLM，模板化评语，confidence=0.95
  └─ 未命中（score = 0）→ 进入第3层
        ↓
      第3层：LLM 兜底（grade_english_fallback）
        ├─ 语义等价命中某采分点 → 按该采分点 score 给分
        ├─ 语义正确但不匹配任何采分点 → 0 分
        └─ 答非所问/拼音/汉字 → 0 分
```

关键原则：**LLM 只能决定"是否命中采分点"，不能创造采分点之外的给分理由。**

## 测试结果

58/59 测试用例通过，覆盖场景：

| 场景 | 示例 | 结果 |
|------|------|------|
| 精确命中高分采分点 | `The Spring Festival` → 2分 | OK |
| 命中低分采分点 | `Spring` → 1分 | OK |
| 排除列表命中 | `New Year` → 0分 | OK |
| CJK 检测 | `春节` → 0分 | OK |
| hit_count 双命中 | `sounds like surplus + yearly prosperity` → 2分 | OK |
| hit_count 单命中 | `sounds like surplus` → 1分 | OK |
| 同义表达命中 | `set off fireworks` → 命中 `light fireworks` | OK |
| 撇号保留 | `Chang'e flew to the moon` → 2分 | OK |
| 答非所问 | `because fish is delicious` → 0分 | OK |
| 3采分点 hit_count | `harmony and patience` (命中2个) → 2分 | OK |

失败的 1 例是测试数据错误（答案拼写不完整），非引擎 bug。

## 已知限制与后续优化

1. **语义放松题依赖 LLM 兜底**：如第 3 题 Q3（嫦娥奔月），精确匹配可能 miss，需 LLM 识别
2. **synonyms 维护**：标点符号如 `/` 会被去除，`september/october` 需写为 `september or october`
3. **多词关键词中间插入词**：`light up fireworks` 不匹配 `light fireworks`，需显式添加到 synonyms
4. **LLM 兜底频次统计**：后续可统计哪些采分点频繁 miss，针对性补充 synonyms
5. **题目 0007-0012**：暂无采分点数据，仍走纯 LLM 评分
