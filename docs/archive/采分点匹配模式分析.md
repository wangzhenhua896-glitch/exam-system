# 采分点匹配模式分析

> 版本：v1.0.0 | 日期：2026-04-13 | 分支：multi-fullscore-answers

## 问题

当前方案（英语采分点评分方案 v1.0.0）中，采分点匹配方式定义为"归一化后的精确子串包含"，使用 Python `in` 操作符：

```python
if keyword in normalized_answer:
    # 命中
```

## 风险：短关键词子串误匹配

纯子串匹配会导致短关键词误命中长词中的子串：

| 采分点关键词 | 学生答案 | 纯子串 `in` | 是否应命中 |
|-------------|---------|-------------|-----------|
| `tea` | `tea` | 匹配 | 是 |
| `tea` | `teapot` | 匹配 | **否** |
| `tea` | `teahouse` | 匹配 | **否** |
| `sun` | `sunset` | 匹配 | **否** |
| `car` | `carbon` | 匹配 | **否** |
| `brick` | `bricklayer` | 匹配 | **否** |

已发现的实际案例：
- 第 6 题 Q1：B 采分点 `tea`（1 分）会误匹配 `teapot`、`teahouse`
- 第 4 题 Q2：B 采分点 `brick` 可能误匹配 `bricklayer`
- 第 4 题 Q2：C 采分点 `earth` 可能误匹配 `earthquake`

## 解决方案：词边界正则匹配

用 `\b` 词边界正则替代纯子串 `in`：

```python
import re
pattern = r'\b' + re.escape(keyword) + r'\b'
if re.search(pattern, normalized_answer):
    # 命中
```

### 行为对比

| 场景 | 示例 | 纯子串 `in` | 词边界 `\b` | 正确行为 |
|------|------|-------------|-------------|---------|
| 独立单词 | `tea` in `a cup of tea` | 匹配 | 匹配 | 匹配 |
| 包含在长词中 | `tea` in `teapot` | 匹配 | **不匹配** | 不匹配 |
| 多词短语 | `green tea` in `a cup of green tea` | 匹配 | 匹配 | 匹配 |
| 标点隔开 | `tea` in `tea, milk` | 匹配 | 匹配 | 匹配 |
| 下划线连接 | `tea` in `tea_pot` | 匹配 | 不匹配 | 不匹配（非自然语言） |

### 为什么 `\b` 是正确的

`\b` 以字母/数字/下划线为词边界，标点和空格都算边界。自然语言中：
- `green tea,` → `tea` 前有空格后有逗号，词边界匹配成功
- `teapot` → `tea` 后紧跟字母 `p`，不是词边界，不匹配

唯一可能误判的是下划线连接（如 `tea_pot`），但这不是自然语言的写法，实际不会遇到。

## 影响范围

此问题影响**所有题目的采分点层**，包括：
- 已录入的第 1-6 题
- 待录入的第 7-12 题
- 未来的任何英语题目

## 决策

**在评分引擎 `_match_scoring_point()` 实现时，统一使用词边界正则匹配，不使用纯子串 `in`。**
