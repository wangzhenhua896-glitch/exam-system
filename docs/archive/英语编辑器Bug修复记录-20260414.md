## Bug 1 + Bug 4: 确认完成按钮 disabled / 进入下一步按钮不出现

### 现象

编辑已有英语题目（含 3 个子题，Q166 Chinese New Year）时：
- Q1 填入标准答案 → 确认完成 → 正常
- Q2/Q3 已有标准答案和采分点 → "确认完成"按钮始终 disabled
- 3 个子题都标记为"已完成"后，"全部小问配置完成，进入下一步"按钮不出现

### 根因

`loadChildren()` 从 DB 加载子题时，每个子题的 `status` 从 `workflow_status` 恢复或默认为 `'pending'`。

模板中编辑区域的显示条件（`question-bank.html:692`）：
```html
<div v-else-if="sq.status === 'in_progress' || sq.expanded">
```

**没有任何子题被设为 `in_progress`**，所以所有非 completed 子题的编辑区域都不可见，"确认完成"按钮自然不可点击。

对比 `extractFromPaste()`（AI 提取后），它会显式设置第一个子题为 `in_progress`：
```javascript
subQuestions.value[0].status = 'in_progress';  // ← extractFromPaste 有这行
```

但 `loadChildren()` 没有等效逻辑。

### 修复

在 `loadChildren()` 末尾，恢复 `activeQuestionIndex` 后，新增逻辑：

```javascript
// 确保活跃题处于 in_progress 状态且展开，其他题折叠
loaded.forEach(function (sq, i) {
  if (i === activeQuestionIndex.value && sq.status !== 'completed') {
    sq.status = 'in_progress';
    sq.expanded = true;
  } else if (sq.status !== 'completed') {
    sq.expanded = false;
  }
});
```

同时增加了数据完整性校验（已有代码，非本次新增）：如果 workflow 认为某题已完成但实际数据不完整（`isQuestionComplete()` 返回 false），则重置为 `pending`。

### 影响范围

- 编辑已有题目（State A, Q166-Q169）时，第一个非完成子题自动展开
- 编辑旧题（State B, Q170-Q177）时同理
- 不影响 AI 提取后的新建流程（`extractFromPaste()` 已有正确逻辑）

## Bug 3: 展开子题后 textarea 累积

### 现象

- Q1 可见 textarea 数量: 1（正常）
- Q2 展开后可见 textarea 数量: 2（异常，Q1 的 textarea 未隐藏）
- Q3 展开后可见 textarea 数量: 3（异常）

### 根因

点击子题进度标签调用 `goToQuestion()` 或完成当前题调用 `completeCurrentQuestion()` 时，只设置了目标题的 `expanded = true`，但没有折叠之前已展开的题目。

`goToQuestion()` 只折叠了 `index` 及之后的题目：
```javascript
for (var i = index; i < subQuestions.value.length; i++) {
  subQuestions.value[i].status = 'pending';
  subQuestions.value[i].expanded = false;
}
// ← index 之前的题（如 Q1）如果被手动展开修改过，expanded 仍为 true
```

### 修复

新增 `collapseAllSubQuestions()` 辅助函数，在导航和完成逻辑中调用：

```javascript
function collapseAllSubQuestions() {
  subQuestions.value.forEach(function (sq) {
    if (sq.status !== 'completed') {
      sq.expanded = false;
    }
  });
}
```

调用点：
1. `completeCurrentQuestion()` — 完成当前题后、展开下一题前，先折叠所有
2. `goToQuestion()` — 跳转到某题前，先折叠所有（在状态重置循环之后）

### 影响范围

- 同一时间只有一个子题的编辑区域可见
- 已完成题的"展开修改"按钮仍可正常展开（手动覆盖折叠）
- 不影响已完成题的摘要显示

## Bug 2: 步骤指示器圆圈未渲染（非 Bug）

### 现象

Playwright 测试查询 `.el-step` 返回空数组，步骤条区域无圆圈指示。

### 分析

步骤指示器在 `question-bank.html:580-602` 中使用**自定义 div 实现**（非 `el-steps` 组件）：

```html
<div style="width: 32px; height: 32px; border-radius: 50%; background: #409eff; ...">
  <span>1</span>
</div>
```

Playwright 测试用 `.el-step` 查找的是 Element Plus Steps 组件的 class，但实际未使用该组件，所以返回空数组是预期行为。

### 结论

代码正确，浏览器中圆圈正常显示。Playwright 测试需要改用正确的选择器（如查询包含步骤文字标签的 div 内的圆形 div）。
