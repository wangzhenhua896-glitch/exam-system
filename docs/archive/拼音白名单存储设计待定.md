# 拼音白名单存储设计待定

> 日期：2026-04-13 | 分支：multi-fullscore-answers

## 问题

部分英语阅读材料中包含拼音借词（如 zongzi, Fu, Xi），这些词在材料原文中出现，学生引用时不应被拼音预检判 0 分。

`three_layer_grader.py` 中 `detect_pinyin()` 支持白名单机制：先移除白名单词汇再检测拼音。白名单通过 `scoring_point_json['pinyin_whitelist']` 读取，是**子题级别**的配置。

## 现状

- 第 2 题（Dragon Boat Festival）已通过白名单处理了 `zongzi`
- 当前白名单存储位置不明确（可能在子题 JSON 中）
- 我们目前录入的 `rubric.points` 是扁平采分点数组，**没有子题分组，也没有白名单字段**

## 需要白名单的题目

| 题号 | 子题 | 拼音借词 |
|------|------|---------|
| 2 | — | zongzi（已有） |
| 11 | Q2 | fu, xi（材料中 `"福" (Fu)` `"喜" (Xi)`） |

## 待定事项

1. **数据结构**：白名单存在哪里？
   - 方案 A：`rubric` JSON 中新增 `sub_questions` 数组，每个子题包含 `pinyin_whitelist`、`score_formula`、`exclude_list`、`scoring_points`
   - 方案 B：`rubric` 中加 `pinyin_whitelist` 顶层字段，共享白名单
   - 方案 C：和 `score_formula` 一起存在 `rubric.points` 的某个元数据字段中

2. **存储位置**：子题级配置（`score_formula`、`exclude_list`、`pinyin_whitelist`）目前散落在文档中，没有结构化存储到数据库。需要设计统一的子题数据结构。

3. **优先级**：引擎实现时再定，文档阶段先记录需求。

## 影响

如果白名单不生效，以下场景会被误判 0 分：
- 第 2 题学生写 `zongzi`（已有白名单保护）
- 第 11 题 Q2 学生写 `Fu` 或 `Xi`（当前无白名单保护）
