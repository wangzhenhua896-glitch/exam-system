## 系统概述

### 设计目标

AI评分系统支持**多个国产大模型服务商**,每个服务商下可配置**多个子模型**。系统需要:

- ✅ 灵活配置: 管理员可独立开关每个服务商和子模型
- ✅ 动态注册: 运行时根据配置自动注册/注销模型实例
- ✅ 透明切换: 评分时可选择任意已启用的模型
- ✅ 易于扩展: 新增服务商/模型只需少量配置和代码

### 支持的服务商

| 服务商 | 标识 | SDK依赖 | 多模型支持 |
|--------|------|---------|-----------|
| 通义千问 (阿里云) | `qwen` | `openai` | ✅ |
| 智谱 GLM | `glm` | `zhipuai` | ✅ |
| MiniMax | `minimax` | `httpx` | ✅ |
| 百度文心 | `ernie` | `qianfan` | ✅ |
| 字节豆包 (火山引擎) | `doubao` | `openai` | ✅ |
| 讯飞星火 | `spark` | (待实现) | ⏳ |
| 小米 Mimimo | `xiaomi_mimimo` | `openai` | ⏳ |

## 三层模型管理

### 核心概念

系统使用**三层结构**管理模型:

```
服务商 (Provider)
  └─ 子模型 (Model)
      └─ 实例 (Instance)
```

| 层级 | 说明 | 配置位置 |
|------|------|---------|
| **服务商** | 如"豆包"、"通义千问" | `.env` 或 数据库 |
| **子模型** | 服务商下的具体模型,如 `deepseek-v3`、`qwen-max` | `available_models` 列表 |
| **实例** | 运行时创建的客户端对象,每个启用的子模型对应一个实例 | `ModelRegistry` 注册表 |

### 配置状态

每个子模型有两个启用状态:

| 状态 | 字段 | 说明 |
|------|------|------|
| **服务商启用** | `enabled` | 整个服务商是否启用 |
| **子模型启用** | `extra_config.enabled_models` | 该服务商下哪些子模型被启用 |

**关系**: 只有服务商 `enabled=true` 且子模型 ID 在 `enabled_models` 列表中,该子模型才会被注册。

## 数据流程

### 1. 配置保存流程

```
管理员在系统管理页面操作
         │
         ▼
前端调用 /api/config/models/{provider}/toggle-model
         │
         ▼
config_routes.py 更新数据库 model_configs.extra_config
         │
         ▼
extra_config.enabled_models = ["model1", "model2", ...]
```

### 2. 模型注册流程

```
应用启动 (routes.py)
         │
         ▼
get_all_effective_configs()
  - 读取 .env 默认配置
  - 读取数据库覆盖配置
  - 合并返回 (包含 available_models 和 extra_config)
         │
         ▼
init_models(config)
  - 遍历每个服务商
  - 读取 available_models 列表
  - 读取 enabled_models 集合
  - 为每个启用的子模型创建客户端实例
  - 注册到 model_registry
         │
         ▼
model_registry.list_models()
  - 返回所有已注册的模型信息
  - 包含 provider, model_name, display_name, enabled
```

### 3. 前端展示流程

```
评分页面加载 (index.html)
         │
         ▼
GET /api/models/available
         │
         ▼
api_routes.available_models()
  - 调用 model_registry.list_models()
  - 返回 JSON: {success: true, models: [...]}
         │
         ▼
前端计算属性 modelGroups
  - 过滤 enabled=true 的模型
  - 按服务商分组
  - 渲染 el-select 下拉框
```

```python
DOUBAO_CONFIG = {
    "api_key": os.getenv("DOUBAO_API_KEY", ""),
    "base_url": os.getenv("DOUBAO_BASE_URL", "..."),

        {"id": "deepseek-v3-2-251201", "name": "DeepSeek-V3.2"},
```

**重要**: `available_models` 定义了该服务商下**所有可选的子模型**。只有在这里声明的模型才能被管理员启用和注册。

### 数据库配置

数据库表 `model_configs` 存储管理员的配置:

```sql
CREATE TABLE model_configs (
    provider TEXT PRIMARY KEY,        -- 服务商标识
    api_key TEXT,                     -- API密钥
    base_url TEXT,                    -- API地址
    model TEXT,                       -- 默认模型
    enabled INTEGER DEFAULT 0,        -- 服务商是否启用
    extra_config TEXT DEFAULT '{}',   -- 额外配置 (JSON)
    updated_at TIMESTAMP
);
```

**extra_config 结构**:

```json
{
    "enabled_models": [
        "deepseek-v3-250324",
        "deepseek-v3-2-251201",
        "doubao-seed-2-0-pro-260215"
    ]
}
```

## 核心模块详解

### 1. 模型注册表 (`app/models/registry.py`)

**职责**: 管理所有运行时模型实例

**关键函数**:

```python
def init_models(config: Dict[str, Any]) -> None:
    """
    初始化所有模型
    
    逻辑:
    1. 遍历每个服务商 (qwen, glm, doubao, ...)
    2. 检查服务商是否启用 (config[provider].enabled)
    3. 读取 available_models 列表
    4. 读取 enabled_models 集合 (extra_config)
    5. 为每个启用的子模型:
       - 创建客户端实例
       - 设置 display_name
       - 注册到 model_registry
    """
    
    # 示例: 豆包多模型注册
    doubao_cfg = config.get("doubao", {})
    if doubao_cfg.get("enabled", False):
        enabled_models = set(doubao_cfg.get("extra_config", {}).get("enabled_models", []))
        available_models = doubao_cfg.get("available_models", [])
        
        for model_info in available_models:
            model_id = model_info["id"]
            # 没有设置 enabled_models → 默认全部启用
            if not enabled_models or model_id in enabled_models:
                model_cfg = {
                    **doubao_cfg,
                    "model": model_id,
                    "display_name": model_info.get("name", model_id),
                }
                client = DoubaoClient(model_cfg)
                model_registry.register(client)
```

**注册表方法**:

| 方法 | 说明 |
|------|------|
| `register(model)` | 注册模型实例 |
| `list_models()` | 返回所有模型信息 (供前端展示) |
| `get_enabled_models()` | 返回所有启用的模型 |
| `get_model_by_provider(provider)` | 根据服务商获取模型 |
| `clear()` | 清空注册表 (用于测试) |

### 2. 配置管理 (`app/models/db_models.py`)

**关键函数**:

```python
def get_effective_config(provider: str) -> dict:
    """
    获取服务商的有效配置 (DB 覆盖 > .env 默认值)
    
    返回:
    {
        'provider': 'doubao',
        'api_key': '从DB或.env获取',
        'base_url': '...',
        'model': 'deepseek-v3-250324',
        'enabled': True,
        'available_models': [...],  # 从 .env 配置中获取
        'extra_config': {           # 从 DB 中解析
            'enabled_models': ['model1', 'model2']
        }
    }
    """
```

**合并逻辑**:

1. 从 `.env` 读取默认配置 (包含 `available_models`)
2. 从数据库读取管理员配置 (包含 `extra_config`)
3. 数据库配置覆盖 `.env` 配置
4. 返回合并后的配置

### 3. 基础客户端类 (`app/models/base.py`)

**所有模型客户端的基类**:

```python
class BaseModelClient(ABC):
    """基础模型客户端"""
    
    provider: ModelProvider
    model_name: str
    enabled: bool = True
    
    def __init__(self, config: Dict[str, Any]):
        self.config = config
        self.enabled = config.get("enabled", True)
    
    @abstractmethod
    async def generate(self, prompt: str, **kwargs) -> ModelResponse:
        """生成响应 (子类实现)"""
        pass
    
    def get_info(self) -> Dict[str, Any]:
        """获取模型信息 (子类可覆盖)"""
        return {
            "provider": self.provider.value,
            "model_name": self.model_name,
            "enabled": self.enabled,
        }
```

**子类必须实现**:

- `__init__(config)`: 初始化客户端,设置 `model_name` 和 `display_name`
- `generate(prompt, **kwargs)`: 调用 API 生成响应
- `get_info()`: 返回模型信息 (包含 `display_name`)

### 4. 前端展示 (`templates/index.html`)

**模型选择下拉框**:

```javascript
// 加载模型列表
const loadModels = async () => {
    const r = await axios.get('/api/models/available');
    if (r.data.success) availableModels.value = r.data.models;
};

// 按服务商分组
const modelGroups = computed(() => {
    const providerLabels = { 
        qwen: '通义千问', 
        doubao: '火山引擎 ARK',
        // ...
    };
    const groups = {};
    // 只显示 enabled=true 的模型
    availableModels.value.filter(m => m.enabled).forEach(m => {
        const p = m.provider || 'other';
        if (!groups[p]) groups[p] = { provider: p, label: providerLabels[p], models: [] };
        groups[p].models.push(m);
    });
    return Object.values(groups);
});
```

**HTML 模板**:

```html

        <el-option v-for="m in group.models" 
                   :key="m.model_name" 
                   :label="m.display_name || m.model_name" 
                   :value="m.model_name">
            <span>{{ m.display_name || m.model_name }}</span>
            <span style="float:right;color:#999;font-size:12px;">{{ m.model_name }}</span>
        </el-option>
    </el-option-group>
```

## 故障排查

### 问题 1: 前端看不到已启用的模型

**症状**: 管理员在系统管理页面启用了模型,但评分页面下拉框中没有

**排查步骤**:

1. 检查数据库配置是否正确保存:
```sql
SELECT provider, enabled, extra_config FROM model_configs WHERE provider='doubao';
```

2. 检查 `extra_config` 中是否有 `enabled_models`:
```json
{"enabled_models": ["model1", "model2"]}
```

3. 检查应用启动日志,看模型是否被正确注册:
```python
# 在 routes.py 添加调试
config = get_all_effective_configs()
print(f"豆包 available_models: {len(config['doubao'].get('available_models', []))}")
print(f"豆包 enabled_models: {config['doubao'].get('extra_config', {}).get('enabled_models', [])}")

init_models(config)
print(f"已注册模型: {[m['model_name'] for m in model_registry.list_models()]}")
```

4. 检查前端 API 调用:
```javascript
// 浏览器控制台
axios.get('/api/models/available').then(r => console.log(r.data.models))
```

**常见原因**:
- ❌ 数据库 `extra_config` 为空或格式错误
- ❌ 应用启动后修改了配置,但没有重启
- ❌ 前端过滤条件 `m.enabled` 为 false

### 问题 2: 模型注册失败

**症状**: 启动时报错 `⚠️ 豆包模型注册失败: ...`

**常见原因**:

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `Missing API Key` | 未配置 API Key | 检查 `.env` 或数据库 |
| `ModuleNotFoundError` | 缺少 SDK | `pip install openai` 等 |
| `Invalid model ID` | 模型 ID 不存在 | 检查 `available_models` 配置 |

### 问题 3: 配置页面看不到子模型列表

**症状**: 系统管理页面编辑服务商时,子模型下拉框为空

**原因**: `available_models` 未配置或格式错误

**解决**:

1. 检查 `config/settings.py` 中该服务商的 `available_models`
2. 确保格式为: `[{"id": "xxx", "name": "xxx"}, ...]`
3. 重启应用
