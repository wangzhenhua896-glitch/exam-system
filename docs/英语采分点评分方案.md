# 英语采分点评分方案

> 版本：v1.0.0 | 日期：2026-04-13 | 分支：multi-fullscore-answers

## 1. 背景与目标

### 现状

英语科目当前评分完全依赖 LLM（第3层），第1层（关键词匹配）和第2层（向量匹配）均跳过。
原因：通用关键词匹配对英语不适用，中文 text2vec 对英文不可靠。

### 问题

- **一致性差**：同一答案不同时间评分可能给不同分（模型输出不确定性）
- **成本高**：每道题都要调一次 LLM API
- **速度慢**：含 3 小问的题目要等 LLM 完整输出

### 目标

将英语评分从"全靠LLM"改为"采分点精确匹配为主 + LLM兜底"：
- 采分点精确命中时，确定性给分，跳过 LLM
- 采分点未命中时，让 LLM 复核
- 大幅减少 API 调用，提升一致性和速度

### 关键约束

**答案均出自原文**：学生的英语答案应直接从阅读材料中摘取，原文拼写正确，
因此采分点匹配只需**精确匹配**（子串包含），不需要模糊匹配/编辑距离容错。
同义表达由文档穷举列出，未列出的不算同义。

## 2. 英语题目结构分析

每题含 3 个小问，每小问满分 2 分，每题总分 6 分。
考试时 3 个小问的答案是**分开提交**的，系统逐小问独立评分（每次 `/api/grade` 只评一个小问）。

### 概念区分：采分点 vs 检查点

- **采分点**（scoring_point）= 一个分值（score）+ 一组检查条件（keywords + synonyms）
- **检查点**（check）= 采分点内部的 keywords/synonyms 中的每个短语，任一命中则该采分点命中

以第 1 题 Q1 为例：

| 采分点 | score | 检查条件（keywords） | 同义表达 |
|--------|-------|---------------------|---------|
| A | 2 | spring festival | — |
| B | 1 | spring, festival | — |

学生答案 `The Spring Festival`，归一化后 `spring festival`：
- 检查 A（2分）：`spring festival` 在答案中 → 命中，得 2 分
- （不再检查 B）

学生答案 `Spring`，归一化后 `spring`：
- 检查 A（2分）：`spring festival` 不在答案中 → 未命中
- 检查 B（1分）：`spring` 在答案中 → 命中，得 1 分

### 两种计分策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| `max_hit_score` | 按 score 从高到低检查，命中第一个就返回 | Q1 型：完整短语=2分，部分=1分 |
| `hit_count` | 逐个检查所有采分点，统计命中数，按规则查表 | Q2 型：3个采分点中命中≥2个=2分 |

## 3. 完整评分规则（逐条提取自 Word 文档）

### 3.1 采分点匹配规则

**统一逻辑**：每个采分点独立匹配，按 score 降序，命中第一个就给分。

#### 第1题（Chinese New Year）

| 小问 | 采分点 | score | 关键词 | 同义表达 |
|------|--------|-------|--------|---------|
| Q1 | A | 2 | spring festival | — |
| Q1 | B | 1 | spring, festival | — |
| Q2 | A | 1 | sounds like surplus | sounds similar to surplus / means surplus |
| Q2 | B | 1 | yearly prosperity | prosperity every year / abundance / more than enough |
| Q3 | A | 1 | light fireworks | set off fireworks / fireworks display / shoot off fireworks |
| Q3 | B | 1 | scare away evil spirits | drive away bad luck / chase away evil / frighten evil spirits |

排除列表（Q1 专属）：`new year`, `lunar new year`

计分方式：Q1 = max_hit_score，Q2 = hit_count（命中2→2分, 命中1→1分），Q3 = hit_count（命中2→2分, 命中1→1分）

排除列表（Q1 专属）：`new year`, `lunar new year`

#### 第2题（Dragon Boat Festival）

| 小问 | 采分点 | 计分方式 | 关键词 | 同义表达 |
|------|--------|---------|--------|---------|
| Q1 | A,B | hit_count: 命中2→2分, 命中1→1分 | A: 5th day of the 5th lunar month / B: June | A: the fifth day of the fifth lunar month / B: in June |
| Q2 | A,B | max_hit_score | A: zongzi / B: rice dumpling | B: sticky rice wrapped in bamboo leaves |
| Q3 | A,B | hit_count: 命中2→2分, 命中1→1分 | A: remember / B: qu yuan | A: honor / commemorate / in memory of |

#### 第3题（The Mid-Autumn Festival）

| 小问 | 采分点 | 计分方式 | 关键词 | 同义表达 |
|------|--------|---------|--------|---------|
| Q1 | A,B | hit_count: 命中2→2分, 命中1→1分 | A: 15th day of the 8th lunar month / B: september or early october | A: the fifteenth day of the eighth lunar month / B: september/october / sept. or early oct. |
| Q2 | A,B,C | hit_count: 命中≥2→2分, 命中1→1分 | A: share mooncakes / B: admire the full moon / C: enjoy time with family | A: eat mooncakes / B: gaze at the moon / C: spend time with family / family reunion |
| Q3 | A,B | max_hit_score | A: chang'e flying to the moon / B: chang'e | A: chang'e flew to the moon / B: the jade rabbit（仅1分） |

注意：第3题 Q3 语义放宽（如 `chang'e flew to the moon`），采分点精确匹配可能未命中，交给 LLM 兜底。

### 3.5 采分点统计总表

| 题号 | 主题 | 小问 | 采分点数 | 计分方式 | 排除列表 | 语义放宽 |
|------|------|------|---------|---------|---------|---------|
| 1 | Chinese New Year | Q1 | 2 | max_hit_score | 有 | 无 |
| 1 | | Q2 | 2 | hit_count | 无 | 无 |
| 1 | | Q3 | 2 | hit_count | 无 | 无 |
| 2 | Dragon Boat Festival | Q1 | 2 | hit_count | 无 | 无 |
| 2 | | Q2 | 2 | max_hit_score | 无 | 无 |
| 2 | | Q3 | 2 | hit_count | 无 | 无 |
| 3 | The Mid-Autumn Festival | Q1 | 2 | hit_count | 无 | 无 |
| 3 | | Q2 | 3 | hit_count | 无 | 无 |
| 3 | | Q3 | 2 | max_hit_score | 无 | 有 |

计分方式分布：
- `max_hit_score`：第1题Q1、第2题Q2、第3题Q3（共3个）
- `hit_count`：其余6个小问

特殊处理：
- 排除列表：仅第1题Q1（new year / lunar new year）
- 语义放宽：仅第3题Q3（chang'e flew to the moon 等，采分点层未命中时交给 LLM）
- 3采分点 hit_count：仅第3题Q2（命中≥2→2分）

### 3.2 直接判 0 分条件（反作弊 + 语言规则）

1. **拼音作答** → 0 分（如 `Chun Jie`）
2. **汉字作答** → 0 分（如 `春节`）
3. **答非所问 / 与材料无关** → 0 分（交给 LLM 兜底判断）

### 3.3 语言宽容度（通用规则，第二步）

4. **大小写不扣分**（`spring festival` = `Spring Festival`）
5. **标点符号不扣分**
6. **冠词不扣分**（`the Spring Festival` = `Spring Festival`）
7. **时态/语态/单复数语法错误不扣分**（只要不影响关键词识别）

### 3.4 冗余信息处理

8. **额外正确信息不扣分**（多写了不扣）

## 4. 数据模型

### 4.1 数据结构

```json
{
  "question_id": 1,
  "sub_questions": [
    {
      "id": "Q1",
      "text": "What is the other name of Chinese New Year?",
      "max_score": 2,
      "scoring_points": [
        {"id": "A", "score": 2, "keywords": ["spring festival"]},
        {"id": "B", "score": 1, "keywords": ["spring", "festival"]}
      ],
      "score_formula": "max_hit_score",
      "exclude_list": ["new year", "lunar new year"]
    },
    {
      "id": "Q2",
      "text": "Why do families serve fish during the reunion dinner?",
      "max_score": 2,
      "scoring_points": [
        {
          "id": "A", "score": 1,
          "keywords": ["sounds like surplus"],
          "synonyms": ["sounds similar to surplus", "means surplus"]
        },
        {
          "id": "B", "score": 1,
          "keywords": ["yearly prosperity"],
          "synonyms": ["prosperity every year", "abundance", "more than enough"]
        }
      ],
      "score_formula": {
        "type": "hit_count",
        "rules": [
          {"min_hits": 2, "score": 2},
          {"min_hits": 1, "score": 1}
        ]
      }
    }
  ]
}
```

每个采分点有 `score` 字段。`max_hit_score` 取最高命中分，`hit_count` 统计命中数按规则查表。

### 4.2 计分公式

| score_formula | 逻辑 | 适用场景 |
|---------------|------|---------|
| `max_hit_score` | 按 score 从高到低检查采分点，命中第一个就返回 | 写出完整短语=2分，部分=1分 |
| `hit_count` | 逐个检查所有采分点，统计命中数，按 rules 查表给分 | 3个采分点中命中≥2个=2分 |

`hit_count` 模式的 score_formula 需要额外的 `rules` 字段：

```json
{
  "score_formula": {
    "type": "hit_count",
    "rules": [
      {"min_hits": 2, "score": 2},
      {"min_hits": 1, "score": 1}
    ]
  }
}
```

采分点按 `score` 从高到低排列，匹配时命中第一个就停止。

### 4.3 文本归一化

匹配前先对答案和采分点做统一归一化：

```
原始文本 → lowercase → 去标点（保留空格）→ 去句首冠词(a/an/the) → 压缩多余空格
```

注意：只去句首冠词，不去句中冠词（避免误伤 `a lot` 等固定搭配）。

### 4.4 采分点匹配模式

统一为 `any` 模式：keywords + synonyms 中的每个表述都是**独立的短语**，任一短语在归一化后的答案中作为子串出现即命中。

匹配方式：**归一化后的精确子串包含**，不做模糊匹配。

示例：
- Q1-A：keywords = `["spring festival"]`，答案中包含 `spring festival` → 命中，得 2 分
- Q1-B：keywords = `["spring", "festival"]`，答案中包含 `spring` 或 `festival` 任一 → 命中，得 1 分
- Q2-A：keywords = `["zongzi"]`，答案中包含 `zongzi` → 命中，得 2 分
- Q3-B：keywords = `["qu yuan"]`，答案中包含 `qu yuan` → 命中

### 4.5 评分逻辑

```
输入：归一化后的学生答案，采分点列表，score_formula

score_formula = max_hit_score 时：
  按 score 从高到低遍历采分点：
    检查该采分点的 keywords + synonyms，任一命中 → 返回该 score
  全部未命中 → 返回 0

score_formula = hit_count 时：
  遍历所有采分点，检查每个的 keywords + synonyms，统计命中数
  按 rules 查表返回对应 score
```

## 5. 评分流程

```
单个小问的学生答案输入
  ↓
【预检】
  ├─ 检测 CJK 字符 → 0 分
  ├─ 检测拼音模式（排除材料中的拼音借词白名单，如 zongzi）→ 0 分
  └─ 空白 / < 2 词 → 0 分
  ↓
【排除列表检查】（如有）
  └─ 答案命中 exclude_list → 0 分
  ↓
【采分点匹配】
  └─ 根据 score_formula 走不同逻辑：
      ├─ max_hit_score：按 score 降序遍历，命中第一个就返回
      └─ hit_count：逐个检查所有采分点，统计命中数，查表给分
  ↓
【出分】
  └─ 返回 score_formula 推导的得分（或 0）
```

## 6. 三层并行流程调整

```
单个小问的学生答案 → 三层并行
  ├─ 第1层：采分点精确匹配（英语专用）→ 得分_A    ← 新增，替代通用关键词层
  ├─ 第2层：向量匹配 → 跳过（英语不适用）
  └─ 第3层：LLM 评分 → 得分_C（保留作为兜底）

  → 如果第1层命中采分点（得分_A > 0），可跳过第3层，直接取得分_A + 模板化评语
  → 如果第1层未命中（得分_A = 0），让第3层 LLM 复核，由 LLM 生成评语
  → 按策略取分（建议 max(A, C)，对学生有利）

### 评语策略

| 场景 | 评语来源 | 示例 |
|------|---------|------|
| 第1层命中 | 模板化，不需要 LLM | "命中 spring festival，得2分。" |
| 第1层未命中 | LLM 生成 | LLM 可能识别出语义等价给分，或判"答案与材料无关，得0分" |
```

## 7. 需要的核心能力

| # | 能力 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | 文本归一化 | P0 | lowercase + 去标点 + 去句首冠词 + 压缩空格 |
| 2 | 精确子串匹配 | P0 | 归一化后子串包含检查，不做模糊匹配 |
| 3 | 采分点内部检查 | P0 | keywords + synonyms，任一短语命中即该采分点命中 |
| 4 | max_hit_score 计分 | P0 | 按 score 降序遍历采分点，命中第一个就返回 |
| 5 | hit_count 计分 | P0 | 逐个检查所有采分点，统计命中数，按规则查表给分 |
| 6 | 同义表达穷举匹配 | P0 | 文档列了就匹配，没列的不算 |
| 7 | 排除列表 | P0 | 命中 exclude_list 直接 0 分 |
| 8 | 中文/拼音检测 | P0 | 直接判 0 分，但排除材料中出现的拼音借词（如 zongzi） |

## 8. 采分点数据来源

### Word 导入时自动提取

Word 文档中的采分点信息结构规整，可在导入时自动解析：

| 行格式 | 提取内容 |
|--------|---------|
| `问题：xxx` | 小问文本 |
| `参考答案：xxx` | 参考答案 |
| `采分点（关键词）：xxx` | 采分点描述 |
| `(A) xxx (B) xxx` | 子采分点拆分（按括号字母） |
| `x分\t条件` | 得分规则 |
| `同义表达可接受范围：` | 同义表达段落开始标记 |
| `xxx → yyy / zzz` | 主表述 → 同义列表 |
| `或答 xxx`（出现在 0 分行） | 排除列表 |
| 阅读材料中的非英文单词 | 拼音借词白名单（如 zongzi, sachets），拼音检测时排除 |

提取后写入 `question_answers` 表，前端导入预览时展示提取结果供老师确认。

## 10. 验证方式

1. 单个小问的各种答案组合能得到正确分数（降序命中即停）
2. 拼音/汉字答案直接 0 分
3. 大小写差异（`spring festival` = `Spring Festival`）正确匹配
4. 冠词差异（`the Spring Festival` = `Spring Festival`）正确匹配
5. 同义表达（如 `set off fireworks`）能正确匹配
6. 排除列表（`New Year` 对 Q1）正确判 0
7. 采分点精确命中时跳过 LLM 调用，模板化评语正确
8. 采分点未命中时触发 LLM 复核
9. 3 个小问分开提交，各自独立评分，互不影响
10. 第2题 Q2 的双层采分点（zongzi=2分 / rice dumpling=1分）降序命中正确
11. 现有思政/语文题目评分不受影响
