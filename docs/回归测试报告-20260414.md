## 关键发现

### 1. 科目决定编辑器类型

`question-bank.html:3302` 的逻辑：
```javascript
if (qItem && qItem.subject === 'english' && !qItem.parent_id) {
    currentPage.value = 'english-edit';
}
```

当前数据库中：
- `subject=general` (Q166-Q171): 有子题，但进**普通编辑器**
- `subject=english` (Q172-Q177): 无子题，进**英语编辑器**（State B 模式）

### 2. Bug 1/3/4 修复代码仍存在

之前的修复（loadChildren 状态初始化、collapseAllSubQuestions、下一步按钮触发）在 englishEditCore.js 中仍然存在，未被覆盖。但因缺少 `subject=english` 且有子题的测试数据，无法通过 UI 自动化复测。

### 3. API 数据完整性良好

- 所有子题 `parent_id` 正确，未出现 NULL 丢失
- `rubric_script` 在 Q191/Q192 上正确写入和读回
- PUT round-trip 通过，数据不丢失
