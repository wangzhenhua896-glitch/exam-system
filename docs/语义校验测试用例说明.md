## 四、语义校验核心逻辑

### 工作流程

```
大模型 API 返回 scoring_items
        │
        ▼
  提取 rubric 中的关键词（_extract_rubric_points）
        │
        ▼
  遍历 hit=False 的要点
        │
        ├── 精确匹配：关键词是否出现在学生答案中？
        │       是 → 纠偏，标记 hit=True
        │
        └── 语义相似度：提取答案片段，与关键词计算余弦相似度
                ≥ 0.72 → 纠偏，标记 hit=True
                < 0.72 → 保持原判
```

### 防误判机制

- **已命中要点排除**：如果关键词是已命中要点文本的子串（如"共同发展"出现在要点1中），跳过精确匹配，避免把要点1的内容误判为要点4
- **异常容错**：语义校验模块抛出任何异常都不影响原始评分结果，仅记录日志

## 五、语义校验测试场景

### 场景 A — V7 近义改写（应纠偏）

**目标**：验证语义校验能发现模型漏判的等价表达。

**模拟输入**：

```
scoring_items:
  要点1: hit=True,  score=2.0  ← 正确
  要点2: hit=True,  score=2.0  ← 正确
  要点3: hit=False, score=0.0  ← 模型漏判（学生用了"民营经济"代替"非公有制经济"）
  要点4: hit=False, score=0.0  ← 模型漏判（学生用了"互相促进"代替"相辅相成"）
```

**预期校验结果**：

```
要点3 → 纠偏
  原因：语义相似度检查
  关键词"非公有制经济"与答案片段"私营经济"相似度 = 0.724 ≥ 阈值 0.72
  → hit=True, score=2.0

要点4 → 纠偏
  原因：精确关键词匹配
  发现关键词"共同发展"出现在学生答案中
  → hit=True, score=2.0

总分: 4.0 → 8.0
```

### 场景 B — V2 确实缺少要点（不应纠偏）

**目标**：验证语义校验不会对真正缺失的内容误纠偏。

**模拟输入**：

```
学生答案中确实没有要点3（非公有制经济）和要点4（相辅相成）的内容。

scoring_items:
  要点1: hit=True,  score=2.0
  要点2: hit=True,  score=2.0
  要点3: hit=False, score=0.0  ← 真的没有
  要点4: hit=False, score=0.0  ← 真的没有
```

**预期校验结果**：

```
要点3 → 不纠偏
  关键词"非公有制经济"不在答案中
  语义相似度最高 0.592 < 0.72
  → 保持 hit=False

要点4 → 不纠偏
  关键词"共同发展"虽在答案中出现，但它是要点1文本"多种所有制经济共同发展"的子串
  触发"已命中要点排除"规则 → 跳过
  语义相似度最高 0.601 < 0.72
  → 保持 hit=False

总分: 4.0 → 4.0（无变化）
```

### 场景 C — V4 空白答案（不应纠偏）

**目标**：验证空白答案不会被误纠偏。

**模拟输入**：

```
学生答案 = ""（空字符串）
全部 4 个要点 hit=False, score=0
```

**预期校验结果**：

```
没有任何关键词匹配
没有可提取的答案片段
全部保持 hit=False, score=0

总分: 0.0 → 0.0（无变化）
```

## 六、语义相似度模型参考数据

### 模型信息

- **模型**: `shibing624/text2vec-base-chinese`
- **类型**: Sentence-BERT（中文语义向量模型）
- **部署**: 本地加载，首次从 hf-mirror.com 下载（约 400MB）
- **计算**: 余弦相似度

### 近义词相似度测试

| 词 A | 词 B | 余弦相似度 | 阈值 0.72 判断 |
|------|------|-----------|--------------|
| 非公有制经济 | 民营经济 | 0.701 | 不通过（接近阈值） |
| 非公有制经济 | 私营经济 | 0.724 | 通过 |
| 非公有制经济 | 国有经济 | 0.549 | 不通过（正确拒绝） |
| 根本特征 | 最根本的特点 | 0.889 | 通过 |
| 重要组成部分 | 不可缺少的重要部分 | 0.869 | 通过 |
| 公有制为主体 | 多种所有制共同发展 | 0.521 | 不通过（正确拒绝） |
| 经济发展 | 天气预报 | 0.285 | 不通过（正确拒绝） |

### 阈值选择说明

- **0.72**：在"民营经济 vs 非公有制经济"（0.701）和"私营经济 vs 非公有制经济"（0.724）之间取折中值
- 低于 0.70 会导致近义词漏判
- 高于 0.75 会导致近义词漏判（原文测试中阈值 0.75 导致"民营经济"无法匹配）

## 七、集成架构

### 代码修改文件

1. **`app/semantic_checker.py`**（新建）— 语义相似度校验模块
2. **`app/qwen_engine.py`**（修改）— 集成到评分引擎

### 集成位置（qwen_engine.py grade() 方法）

```
大模型 API 调用
    │
    ▼
_parse_output()
    解析模型输出，提取 scoring_items
    │
    ▼
格式校验（validated_items）
    逐项提取 name/score/max_score/hit/reason/quoted_text
    │
    ▼
_extract_rubric_points(rubric)          ← 新增
    从 rubric 字典中提取结构化要点和关键词
    优先级: points(JSON) > rubricScript > rubricPoints
    │
    ▼
validate_scoring_items(                 ← 新增
    answer, scoring_items, rubric_points,
    similarity_threshold=0.72
)
    对 hit=False 的要点做二次校验
    │
    ├── 有纠偏 → 重新累加 final_score，写入 warning
    └── 无纠偏 → 保持原评分
    │
    ▼
_calculate_confidence()
    │
    ▼
boundary_check()
    │
    ▼
返回 QwenGradingResult
```

### 关键方法说明

**`_extract_rubric_points(rubric)`** — 从三种格式中提取关键词：

| 优先级 | 来源 | 格式示例 |
|--------|------|---------|
| 1 | `points` (JSON) | `[{"description": "...", "keywords": ["kw1", "kw2"]}]` |
| 2 | `rubricScript` | `要点1（2分）：公有制为主体、多种所有制经济共同发展` |
| 3 | `rubricPoints` | `公有制为主体、多种所有制经济共同发展 (2分)` |

**`_keywords_from_text(text)`** — 从文本中自动提取关键词：

- 按顿号、逗号、"和"、"及" 切分
- 过滤长度：只保留 2~12 字的片段
- 兜底：切分后不足 2 个关键词时，把整句（截取前 30 字）也加入

## 八、已知限制

1. **近义词识别不完美**：text2vec 模型对某些领域术语（如"民营经济 vs 非公有制经济"）相似度仅为 0.701，低于 0.72 阈值
2. **关键词提取依赖文本格式**：如果要点文本没有顿号、逗号等分隔符，只能以整句作为关键词
3. **无法纠偏 hit=True 的误判**：语义校验只处理模型标记为未命中的要点，对 hit=True 但给分错误的情况无能为力
4. **性能开销**：首次加载 text2vec 模型约 10 秒（后续缓存），每次校验增加约 0.5 秒延迟
