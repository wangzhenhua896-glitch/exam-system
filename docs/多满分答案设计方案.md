# 多满分答案支持方案

> 版本：v1.0.0 | 日期：2026-04-12 | 分支：multi-fullscore-answers

## 1. 背景与目标

### 问题

当前系统每道题只有一个 `standard_answer`（单行 TEXT 字段）。但简答题天然存在"多个满分答案"的情况——不同学生可以用完全不同的表述拿到满分。

### 目标

- 存储每道题的多个满分答案（整题级 + 采分点级）
- 满分答案库作为评分引擎的第一层：查表直接打分，命中则跳过 LLM
- 支持所有科目：思政、语文、英语
- 支持父子题结构（材料+多道小题），兼容英语阅读理解和未来思政材料题
- 三层并行评分 + 管理员可配置取分策略

## 2. 数据库改动

### 2.1 questions 表新增字段

```sql
ALTER TABLE questions ADD COLUMN parent_id INTEGER DEFAULT NULL REFERENCES questions(id);
ALTER TABLE questions ADD COLUMN scoring_strategy TEXT DEFAULT NULL;
CREATE INDEX idx_questions_parent ON questions(parent_id);
```

| 字段 | 说明 |
|------|------|
| parent_id | 父题 ID，NULL 表示独立题，非 NULL 表示子题 |
| scoring_strategy | 单题评分策略覆盖：max/min/avg/median，NULL 使用全局默认 |

### 2.2 新增 question_answers 表

```sql
CREATE TABLE IF NOT EXISTS question_answers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id INTEGER NOT NULL REFERENCES questions(id) ON DELETE CASCADE,
    scope_type TEXT NOT NULL DEFAULT 'question',  -- question / scoring_point
    scope_id TEXT DEFAULT '',       -- 采分点编号：自增序号 "1"、"2"
    score_ratio REAL DEFAULT 1.0,   -- 占该采分点满分的百分比（1.0=满分，0.5=半分）
    answer_text TEXT NOT NULL,       -- 答案文本（用于匹配度计算）
    label TEXT DEFAULT '',           -- 标注：如"满分标准表述"、"Spring Festival"
    source TEXT DEFAULT 'manual',    -- 来源：manual / ai_generated / student_real
    sort_order INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_qa_question ON question_answers(question_id, scope_type);
```

**scope_type 只有两个值：**
- `'question'` — 整题级满分答案（独立题或子题的整体答案）
- `'scoring_point'` — 采分点级等价表述

**scope_id 编号规则：**
- 子题/题内自增序号（1, 2, 3...），纯数字，不嵌套
- 前端是源头：老师录入时给每个答案标注采分点，label 存显示名称
- 评分脚本反过来从数据生成：取 scope_id + label + answer_text

### 2.3 新增 grading_params 表

```sql
CREATE TABLE IF NOT EXISTS grading_params (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL,
    description TEXT DEFAULT '',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO grading_params (key, value, description) VALUES
('match_pass_threshold', '0.80', '学生答案与满分答案的匹配度高于此值，直接判定得分，不再调用AI'),
('match_review_threshold', '0.65', '匹配度在此区间，交给AI复核后给分'),
('full_check_threshold', '0.72', '整题评分后的校验阈值，偏离此值触发复查'),

```

### 2.4 迁移策略

- `questions.standard_answer` → 迁移到 `question_answers`（scope_type='question', score_ratio=1.0, label='标准答案'）
- 保留原字段不删除（兼容旧代码）
- grading_params 插入 4 个默认阈值

## 3. 父子题结构

```
questions 表：
  id=37, parent_id=NULL  → 独立题（思政简答题）
  id=50, parent_id=NULL  → 材料题（英语阅读，含阅读材料在 content 中）
  id=51, parent_id=50    → 子题（英语第1小题）
  id=52, parent_id=50    → 子题（英语第2小题）
  id=53, parent_id=50    → 子题（英语第3小题）
```

- **独立题**：思政简答题、语文简答题（当前已有）
- **父题**：含阅读材料的壳，`content` 存材料全文，`max_score` = 各子题之和
- **子题**：独立的 questions 行，有自己的 `content`（问题文本）、`max_score`、`rubric_script`

**父子题满分值校验（三环节）：**
- 保存时：`parent.max_score == sum(children.max_score)`，不等则拒绝保存
- 展示时：题库页面展示父题时，总分不一致显示黄色警告
- 评分时：总分不一致标记 needs_review 并记录 bug_log

## 4. 评分流程（三层并行 + 策略取分）

```
学生答案 → 三层并行执行
  ├─ 第1层：关键词匹配 → 得分_A
  ├─ 第2层：向量匹配度 → 得分_B
  └─ 第3层：LLM评分 → 得分_C

  → 读取 scoring_strategy（questions 单题覆盖 → grading_params 全局默认）
  → 按策略计算最终得分
  → 前端展示三层分值 + 最终得分
```

### 取分策略

| 策略 | 计算方式 |
|------|---------|
| max | max(A, B, C) |
| min | min(A, B, C) |
| avg | (A + B + C) / 3 |
| median | 去掉最高最低取中间 |

### 反义关键词处理

关键词只做候选筛选，不直接给分。关键词命中后由向量层确认：
- 正向匹配（"公有制为主体"）→ 向量相似度高（≥ 0.80）→ 确认给分
- 反义匹配（"不以公有制为主体"）→ 向量相似度低（< 0.3）→ 自动过滤

## 5. API 端点

### 满分答案管理

```
GET    /api/questions/<id>/answers?scope_type=question  -- 获取某题的满分答案
POST   /api/questions/<id>/answers      -- 追加一个满分答案
PUT    /api/questions/<id>/answers/<aid> -- 修改某个答案
DELETE /api/questions/<id>/answers/<aid> -- 删除某个答案
```

### 阈值配置

```
GET    /api/grading-params              -- 获取所有阈值配置
PUT    /api/grading-params/<key>        -- 修改某个阈值
```

### 父子题

```
GET    /api/questions/<id>/children     -- 获取某题的子题列表
POST   /api/grade-batch                 -- 批量评分子题（传入 parent_id）
```
